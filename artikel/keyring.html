🔐 Ubuntu Security Notes

KeyRing: Menyimpan Kata Sandi di Ubuntu

KeyRing adalah komponen keamanan di Ubuntu dan lingkungan desktop Linux yang bertugas menyimpan kata sandi secara aman. Ia biasa dipakai untuk menyimpan akses jaringan nirkabel, SSH, sertifikat, dan kredensial aplikasi lain agar pengguna tidak perlu mengetik ulang kata sandi setiap saat.

Apa Fungsi KeyRing?

Sederhananya, KeyRing adalah brankas kecil di sistem Linux. Ketika aplikasi membutuhkan kata sandi, misalnya untuk jaringan Wi-Fi, koneksi SSH, atau kredensial tertentu, KeyRing dapat menyimpannya secara terenkripsi.

Di banyak distro Linux, aplikasi pengelola KeyRing biasanya bisa dibuka lewat aplikasi bernama seahorse. Dari sana, pengguna dapat melihat daftar kunci, sertifikat, dan kata sandi yang tersimpan.

Catatan: KeyRing bukan sekadar pengingat kata sandi. Ia adalah bagian dari mekanisme penyimpanan rahasia sistem. Jadi, mematikannya sebaiknya dilakukan hanya jika memang benar-benar memahami risikonya.

Mematikan KeyRing Saat Startup

Pada beberapa kasus, pengguna merasa terganggu karena KeyRing terus meminta kata sandi saat sistem dinyalakan. Jika ingin menonaktifkan layanan tertentu, buka:

Menu → System → Preferences → Startup Applications

Lalu hapus centang pada layanan berikut:

  • Certificate and Keyring Storage
  • Secret Storage Service
  • SSH Key Agent

Jika Masih Bermasalah

Jika KeyRing masih tetap muncul atau konfigurasi lama bermasalah, kamu bisa menghapus file keyring lama melalui terminal. 

cd .gnome2/keyrings/
sudo rm *.keyring
sudo reboot
Peringatan: Perintah di atas dapat menghapus data keyring lama. Pastikan kamu paham konsekuensinya, terutama jika ada kredensial penting yang masih tersimpan di sana.
