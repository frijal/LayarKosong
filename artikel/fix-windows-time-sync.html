🕒 Cara Memperbaiki Masalah Sinkronisasi Waktu di Windows 10/11 (Layanan Waktu & Pengaturan NTP)

Durasi: 15 menit Tingkat: Pemula Windows 10 / 11 Diperbarui: Maret 2026

Pernahkah jam di laptop atau PC kamu tiba-tiba meleset beberapa menit, bahkan hingga hitungan jam? Atau saat mencoba sinkronisasi waktu di pengaturan, muncul error "Time synchronization failed"? Jangan panik! 🧘♂️ Masalah ini lebih sering terjadi daripada yang dibayangkan, dan kabar baiknya: hampir selalu bisa diperbaiki tanpa harus bawa ke tukang servis.

Waktu sistem yang salah bisa mengacaukan banyak hal: sertifikat keamanan website jadi error, akun Microsoft susah login, update Windows gagal, bahkan jadwal backup atau task otomatis berantakan. Di artikel ini, kita akan bahas tuntas dari akar masalah – mulai dari cek zona waktu, memastikan Windows Time service berjalan, hingga mengatur ulang NTP server biar jam komputer kembali akurat.

📋 Prasyarat: Pastikan PC terhubung internet, kamu punya hak administrator, zona waktu sudah benar, serta pakai Windows 10 atau 11. (Tampilan antarmuka sedikit berbeda antar versi, tapi opsi intinya mirip).

🔍 Mengapa Jam Windows Bisa Salah? (5W+1H)

Secara default, Windows menyinkronkan waktu melalui internet dengan server NTP (Network Time Protocol). Namun layanan Windows Time bisa mati, server default time.windows.com kadang tidak merespon, atau pengaturan CMOS baterai melemah. Belum lagi jika PC tergabung dalam domain kantor – maka sinkronisasi dikontrol server perusahaan. Pahami dulu akar masalah biar solusi tepat sasaran!

Langkah 1: Periksa Zona Waktu & Pengaturan Dasar

Sebelum ambil pusing, pastikan zona waktu sudah sesuai. Tekan Windows + I untuk buka Settings → Time & Language → *Date & time*. Di Windows 11, masuk ke *Time & language > Date & time*. Aktifkan opsi Set time automatically dan Set time zone automatically jika tersedia. Pastikan zona waktu yang muncul sesuai daerahmu. Jika deteksi otomatis salah, nonaktifkan lalu pilih zona waktu secara manual.

Tip: Zona waktu yang salah bisa membuat jam tampak meleset meskipun sinkronisasi internet berhasil.

Langkah 2: Sinkronisasi Manual Lewat Settings

Di halaman Date & time yang sama, scroll ke bawah hingga menemukan Additional settings atau *Synchronize your clock*. Klik tombol Sync now. Jika berhasil, akan muncul keterangan waktu sinkron terakhir. Kalau gagal (muncul error), lanjut ke langkah berikutnya.

Langkah 3: Pastikan Layanan Windows Time Berjalan

Windows punya service bernama Windows Time (W32Time) yang bertanggung jawab atas sinkronisasi jam. Tekan Windows + R, ketik services.msc, lalu Enter*. Di jendela Services, cari Windows Time. Double‑klik, atur *Startup type menjadi Automatic atau Automatic (Trigger Start), dan pastikan Service status bertuliskan Running. Jika tidak, klik Start. Bila perlu, klik Stop lalu Start lagi untuk me-restart service.

Peringatan: Jika layanan Windows Time dinonaktifkan, tombol Sync di Settings tidak akan berfungsi optimal.
Langkah 4: Paksa Sinkronisasi Ulang via Command Prompt (Admin)

Jika langkah sebelumnya belum mempan, kita panggil kekuatan command line. Buka Command Prompt sebagai Administrator (klik kanan Start → Terminal Admin atau *Command Prompt Admin). Jalankan perintah ini satu per satu:

net stop w32time
w32tm /unregister
w32tm /register
net start w32time
w32tm /resync

Setelah perintah terakhir, jika muncul pesan The command completed successfully. artinya sync berhasil. Jika gagal, jangan khawatir – lanjut ke step berikutnya untuk mengganti server NTP.

Langkah 5: Konfigurasi Server NTP yang Andal

Kadang server default time.windows.com sedang sibuk atau tidak stabil. Kita bisa mengganti ke server publik seperti pool.ntp.org.

Opsi A (via Control Panel): Tekan Windows + R, ketik control, buka Clock and Region > Date and Time → tab Internet Time → Change settings*. Centang *Synchronize with an Internet time server*, lalu isi Server: time.windows.com atau pool.ntp.org, klik *Update now → OK.

Opsi B (Command Prompt - lebih kuat): Jalankan sebagai admin:

w32tm /config /manualpeerlist:"time.windows.com,0x8 pool.ntp.org,0x8" /syncfromflags:manual /update
net stop w32time
net start w32time
w32tm /resync
*Catatan: `pool.ntp.org` adalah kumpulan server NTP publik yang sangat andal dan sering dipakai jika server default bermasalah.*
Langkah 6: Periksa Apakah PC Tergabung Domain Kantor

Jika laptop/PC adalah perangkat kantor atau sekolah yang terhubung ke domain, kebijakan waktu biasanya diatur oleh server pusat (Domain Controller). Kamu tidak bisa mengubahnya secara manual. Cek dengan masuk ke Settings → System → About*, lihat bagian *Domain atau *Workgroup*. Jika tertulis domain, sebaiknya hubungi tim IT sebelum mengubah pengaturan waktu.

Langkah 7: Verifikasi Status Sinkronisasi

Setelah semua perubahan, pastikan semuanya berfungsi. Buka Command Prompt (admin) dan ketik:

w32tm /query /status

Perhatikan baris Source (harusnya menunjuk NTP server seperti time.windows.com,0x8) dan Last Successful Sync Time (waktu terakhir sinkron). Jika masih belum berubah, coba jalankan ulang w32tm /resync atau restart PC.

Langkah 8: Restart Komputer & Uji Coba

Beberapa perubahan baru akan efektif setelah reboot. Simpan pekerjaan, lalu restart PC. Setelah masuk kembali ke Windows, buka Settings > Date & time, lalu klik Sync now untuk memastikan jam tetap akurat. Dalam beberapa menit, Windows akan terus menyinkron secara periodik.

🛠️ Jika Sinkronisasi Masih Gagal? Cek Daftar Ini

Koneksi internet stabil? Pastikan tidak sedang pakai VPN/proxy yang memblokir NTP (port 123 UDP). Coba matikan sementara firewall atau antivirus pihak ketiga untuk uji coba.
CMOS Battery lemah? Jika jam selalu meleset setiap kali komputer dimatikan total (tanpa listrik), kemungkinan baterai CMOS di motherboard sudah habis. Ganti baterai CMOS bisa jadi solusi permanen.
Tombol Sync abu-abu atau hilang? Bisa karena kebijakan grup atau akun bukan administrator. Coba langkah via Command Prompt saja, atau minta bantuan admin perangkat.

✨ Kesimpulan & Intisari

Masalah waktu di Windows 10/11 sebenarnya mudah diatasi jika tahu jalurnya. Mulai dari pengecekan zona waktu, restart layanan Windows Time, hingga mengganti server NTP menjadi pool.ntp.org. Dengan jam yang akurat, pengalaman browsing, login aplikasi, dan keamanan sistem akan jauh lebih lancar. Jangan sampai masalah sepele ini mengganggu produktivitasmu.

Takeaways Penting:
✔ Waktu sistem yang salah bisa memicu error sertifikat, gagal update, dan masalah login.
✔ Layanan Windows Time perlu berjalan (Startup type: Automatic).
✔ Gunakan perintah `w32tm /resync` untuk memaksa sinkron ulang.
✔ Server NTP alternatif (`pool.ntp.org`) sering lebih responsif.
✔ Jika jam kacau setiap shutdown, kemungkinan baterai CMOS lemah.
