Skrip fedora-mirror.sh: Aktifkan Mirror Tercepat

Pernahkah Anda duduk memandangi terminal, menunggu proses instalasi paket di Fedora Linux selesai, sambil bertanya-tanya mengapa kecepatannya hanya mentok di beberapa kilobyte per detik? Apalagi jika Anda sedang bersantai menikmati kopi di Balikpapan, rasanya waktu terbuang sia-sia hanya untuk mengunduh pembaruan sistem yang seharusnya bisa secepat kilat.

Masalah ini sebenarnya cukup klasik. Secara bawaan, Fedora menggunakan dnf sebagai manajer paket andalannya. Namun, sistem sering kali secara buta mengunduh dari server mirror yang letak geografisnya jauh dari lokasi Anda, tanpa memperhitungkan latensi dan bandwidth yang tersedia. Padahal, untuk mempercepat update dan instalasi, kita hanya perlu mengaktifkan plugin bawaan yang sangat cerdas: fastestmirror.

Mengapa Anda Membutuhkan fastestmirror?

Plugin fastestmirror bekerja dengan cara yang elegan. Saat Anda menjalankan perintah pembaruan atau instalasi, plugin ini akan secara proaktif mengukur waktu respons koneksi dari daftar cermin publik Fedora. Ia kemudian menyeleksi dan otomatis memilih server dengan ping terendah, memastikan aliran data yang masuk ke komputer Anda berada pada kecepatan optimal. Ini adalah solusi "sekali atur, lupakan" yang disarankan diterapkan oleh setiap administrator maupun pengguna harian Fedora. Hasil dapat berbeda tergantung perangkat, versi paket, dan konfigurasi sistem.

⚠️ Peringatan Akses: Jalankan skrip di bawah ini dengan mode sudo (Superuser). Kita perlu merombak dan menulis ulang konfigurasi inti manajer paket yang bersarang di dalam direktori sistem /etc/dnf/dnf.conf.

Automasi Lewat Skrip Bash yang Rapi

Daripada membuka teks editor dan mengetik konfigurasi secara manual yang rentan mengalami salah ketik (typo), mari kita buat proses ini sepenuhnya otomatis. Skrip bash sederhana di bawah ini bernama fedora-mirror.sh. Ia tidak hanya menyuntikkan baris konfigurasi yang dibutuhkan, tetapi juga memiliki inisiatif tinggi untuk mem-backup konfigurasi lama Anda demi keamanan, serta menyapu bersih cache DNF agar pengaturan baru segera berlaku.

Salin kode berikut, simpan sebagai file fedora-mirror.sh, berikan izin eksekusi menggunakan perintah chmod +x fedora-mirror.sh, dan jalankan. 

#!/bin/bash
# fedora-mirror.sh - Mengaktifkan mirror DNF tercepat di Fedora
set -e
echo "[INFO] Mem-backup konfigurasi DNF..."
sudo cp /etc/dnf/dnf.conf /etc/dnf/dnf.conf.bak
echo "[INFO] Mengaktifkan 'fastestmirror' di /etc/dnf/dnf.conf..."
if ! grep -q "fastestmirror=True" /etc/dnf/dnf.conf; then
    echo "fastestmirror=True" | sudo tee -a /etc/dnf/dnf.conf
fi
echo "[INFO] Membersihkan cache DNF..."
sudo dnf clean all
echo "[SELESAI] Plugin 'fastestmirror' berhasil diaktifkan!"

Setelah skrip di atas menampilkan pesan sukses, cobalah lakukan sudo dnf update. Anda akan langsung merasakan perbedaan aliran data yang lebih gesit dari sebelumnya, karena sistem kini mengambil paket langsung dari server terdekat yang sehat. Hasil dapat berbeda tergantung perangkat, versi paket, dan konfigurasi sistem. Selamat tinggal unduhan lelet, dan selamat mengoprek!
