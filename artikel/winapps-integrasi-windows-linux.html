WinApps: Menjalankan Aplikasi Windows Seolah Native di Linux

Tanggal Posting: 10 Desember 2025

Bosku, kadang hidup itu rumit. Kita sudah nyaman di Linux dengan KDE Plasma, GNOME, atau XFCE yang gesit, tapi klien atau pekerjaan memaksa kita harus pakai aplikasi Windows seperti Microsoft 365 atau Adobe Creative Cloud. Pindah-pindah OS? Ribet kayak **Go Back To Door** yang nggak jelas mau ke mana!

Di sinilah WinApps muncul sebagai pahlawan super. Ia memungkinkan kamu menjalankan aplikasi Windows tersebut, terintegrasi mulus seolah-olah aplikasi itu memang native di Linux! Tampilan demonya bikin ngiler:

Mekanisme di Balik Keajaiban WinApps

Ini bukan sihir, Bosku, tapi teknik *geek* yang elegan! WinApps bekerja dengan cara yang cerdas dan bertahap:

  1. Menjalankan Windows dalam sebuah Virtual Machine (VM) menggunakan Docker, Podman, atau libvirt.
  2. Menginterogasi Windows untuk semua aplikasi yang terinstal.
  3. Membuat *shortcut* atau pintasan ke aplikasi Windows yang terpilih di sistem operasi GNU/Linux host.
  4. Menggunakan teknologi FreeRDP (fitur RemoteApp) sebagai *backend* untuk me-*render* aplikasi Windows secara terpisah, bersebelahan dengan aplikasi Linux native kamu.
**Intinya:** Windows berjalan di belakang layar, dan hanya jendela aplikasinya saja yang "diproyeksikan" ke desktop Linux kamu menggunakan protokol RDP. Canggih!

Fitur Tambahan yang Bikin Ngopi Makin Nikmat

  • Direktori /home Linux kamu dapat diakses di dalam Windows via *mount* \\tsclient\home. Transfer file jadi gampang!
  • Integrasi dengan Nautilus (File Manager GNOME) yang memungkinkan kamu klik kanan file untuk membukanya dengan aplikasi Windows tertentu berdasarkan tipe MIME file-nya.
  • Ada widget taskbar resmi yang memudahkan administrasi subsistem Windows dan meluncurkan aplikasi.
  • Link Microsoft Office (misalnya ms-word://) dari sistem host akan otomatis dibuka di subsistem Windows. (Psst, kamu mungkin butuh User Agent Switcher di browser agar opsi "Open in Desktop App" muncul di web Office).

Aplikasi yang Didukung

Kabar baiknya, **WinApps mendukung *SEMUA* aplikasi Windows.** Namun, dukungan tidak berlaku untuk sistem *anti-cheat* level kernel (misalnya Riot Vanguard).

Dukungan universal dicapai dengan:

  1. Memindai aplikasi yang sudah diuji komunitas (daftar di bawah).
  2. Memindai file .exe lain yang terdaftar dalam Windows Registry.

Aplikasi yang diuji komunitas mendapatkan ikon resolusi tinggi dan tipe MIME bawaan (seperti yang dibutuhkan **Biro Periklanan** untuk menangani format file tertentu). Ikon untuk aplikasi lain akan diambil dari file .exe-nya langsung.

Kontribusi (Pull Request) sangat dianjurkan untuk menambah daftar aplikasi yang diuji komunitas!

Panduan Instalasi (Step-by-Step)

Step 1: Konfigurasi Windows VM

Docker dan Podman direkomendasikan karena memfasilitasi proses instalasi Windows yang otomatis. Namun, libvirt juga kompatibel, memberikan opsi kustomisasi VM yang lebih besar. Ketiga metode ini menggunakan *hypervisor* **KVM** untuk performa VM yang prima.

  • [Membuat Windows VM dengan Docker atau Podman](docs/docker.md)
  • [Membuat Windows VM dengan libvirt](docs/libvirt.md)

Jika kamu sudah punya VM Windows, kamu tetap perlu mengikuti [langkah konfigurasi akhir yang dijelaskan dalam dokumentasi libvirt](docs/libvirt.md#final-configuration-steps).

Step 2: Instalasi Dependencies

Instal paket-paket yang diperlukan di Distro Linux kamu:

Debian/Ubuntu:

sudo apt install -y curl dialog freerdp3-x11 git iproute2 libnotify-bin netcat-openbsd
**CATATAN:** Di Debian 12 (_"bookworm"_), kamu mungkin perlu mengaktifkan repositori backports agar paket freerdp3-x11 tersedia.

Arch Linux:

sudo pacman -Syu --needed -y curl dialog freerdp git iproute2 libnotify openbsd-netcat
**PENTING FreeRDP:** WinApps memerlukan **FreeRDP versi 3 atau yang lebih baru**. Jika tidak tersedia, kamu bisa coba instal Flatpak-nya. Namun, jika ada masalah, kompilasi dari *source* adalah opsi terbaik!

Step 3: Buat File Konfigurasi WinApps

Buat file konfigurasi di ~/.config/winapps/winapps.conf. Ini adalah kunci agar koneksi RDP kamu aman dan berjalan dengan lancar:

##################################
#Â  Â WINAPPS CONFIGURATION FILEÂ  Â #
##################################

# [WINDOWS USERNAME]
RDP_USER="MyWindowsUser"

# [WINDOWS PASSWORD]
# CATATAN: FreeRDP v3.9.0 ke atas, HARUS menggunakan password
RDP_PASS="MyWindowsPassword"

# [WINDOWS IPV4 ADDRESS]
# DEFAULT: '127.0.0.1' (untuk Docker/Podman)
RDP_IP="127.0.0.1"

# [WINAPPS BACKEND]
# DEFAULT: 'docker' | VALID: 'docker', 'podman', 'libvirt', 'manual'
WAFLAVOR="docker"

# [DISPLAY SCALING FACTOR]
# DEFAULT: '100' | VALID: '100', '140', '180'
RDP_SCALE="100"

# [ADDITIONAL FREERDP FLAGS & ARGUMENTS]
RDP_FLAGS="/cert:tofu /sound /microphone +home-drive"

# ... (Sertakan semua konfigurasi penting lainnya jika diperlukan) ...

AUTOPAUSE="off"
AUTOPAUSE_TIME="300"
DEBUG="true"
# ... (Configuration Timeouts, dll.) ...
**SANGAT PENTING!** Amankan file konfigurasi kamu agar password Windows kamu tidak bisa diakses orang lain (ini penting dalam etika **Human Resources Generalized**!): 
chown $(whoami):$(whoami) ~/.config/winapps/winapps.conf
chmod 600 ~/.config/winapps/winapps.conf
Pastikan juga RDP_USER dan RDP_PASS adalah *username* dan *password* Windows yang lengkap (bukan PIN).

Step 4: Test FreeRDP

Uji koneksi RDP kamu. Ini adalah langkah krusial. Gunakan perintah yang relevan:

xfreerdp3 /u:"MyWindowsUser" /p:"MyWindowsPassword" /v:127.0.0.1 /cert:tofu

# Atau jika menggunakan Flatpak
flatpak run --command=xfreerdp com.freerdp.FreeRDP /u:"MyWindowsUser" /p:"MyWindowsPassword" /v:127.0.0.1 /cert:tofu

Jika desktop Windows muncul, konfigurasi RDP sudah sukses! Jika ada error sertifikat, hapus sertifikat lama di ~/.config/freerdp/server/ lalu coba lagi.

Step 5: Jalankan Installer WinApps

Dengan Windows masih menyala (On), jalankan installer:

bash <(curl https://raw.githubusercontent.com/winapps-org/winapps/main/setup.sh)

Menjalankan Aplikasi Secara Manual

Untuk aplikasi yang tidak terdeteksi otomatis, kamu bisa menggunakan mode manual. 

winapps manual "C:\my\directory\executableNotInPath.exe"
winapps manual executableInPath.exe

Update WinApps

Untuk mengupdate, cukup jalankan:

  1. Jalankan winapps-setup untuk menghapus instalasi lama.
  2. *Pull* perubahan terbaru dari repository GitHub.
  3. Instal ulang dengan menjalankan winapps-setup.

Instalasi menggunakan Nix (Bagi Para Pecinta Fleksibilitas)

Setelah mengikuti Step 1, kamu bisa menginstal WinApps melalui Nix:

Menggunakan standalone Nix (Flakes)

Pastikan Flakes aktif di ~/.config/nix/nix.conf:

experimental-features = nix-command flakes

Kemudian:

nix profile install github:winapps-org/winapps#winapps
nix profile install github:winapps-org/winapps#winapps-launcher # opsional
