⚡ Memilih Mirror Tercepat di Slackware & NixOS

Sysadmin Tips Oleh: Tim Kontributor Linux Publikasi: Juni 2026

Pernahkah Anda merasa kesal saat melakukan pembaruan sistem operasi Linux namun kecepatan unduhannya merayap seperti siput? Masalah ini sangat umum terjadi ketika sistem kita secara bawaan mengarah ke server repositori utama global yang jarak geografisnya sangat jauh dari lokasi kita saat ini. Solusi paling jitu untuk mengatasinya adalah dengan melakukan kustomisasi server cermin atau yang biasa kita sebut sebagai mirror.

Optimasi jalur unduhan ini tidak hanya menghemat waktu, tetapi juga memastikan kestabilan transfer data saat melakukan pembongkaran paket besar. Pada artikel kali ini, kita akan membahas secara mendalam bagaimana langkah taktis dalam mengatur dan memilih mirror tercepat untuk dua distribusi Linux yang memiliki karakteristik unik: Slackware dengan pendekatan tradisionalnya yang legendaris, serta NixOS dengan paradigma deklaratif modernnya yang revolusioner.

1. Slackware (slackpkg)

Slackware, sebagai salah satu distro Linux tertua yang masih aktif dikembangkan, mengandalkan kesederhanaan sistem penanganan paketnya. Distro ini menggunakan manajer paket bawaan bernama slackpkg. Secara bawaan, instalasi baru cenderung tidak mengaktifkan mirror satu pun demi keamanan. Hasil dapat berbeda tergantung perangkat, versi paket, dan konfigurasi sistem. Tugas kita sebagai pengguna adalah menentukan dan membuka segel konfigurasi tersebut agar pembaruan paket dapat berjalan.

Untuk memilih mirror tercepat dan terdekat dari lokasi kita, langkah awal yang harus dilakukan adalah memodifikasi berkas teks konfigurasi utama yang terletak di jalur /etc/slackpkg/mirrors. Di dalam berkas ini, Anda akan disuguhi ratusan daftar server dari berbagai belahan dunia yang diurutkan berdasarkan negara.

# Buka daftar mirror Slackware
sudo nano /etc/slackpkg/mirrors
# Contoh: aktifkan mirror Indonesia (hapus tanda # di depan)
ftp://kartolo.sby.datautama.net.id/slackware/slackware64-15.0/
# Simpan lalu update database slackpkg
sudo slackpkg update gpg
sudo slackpkg update

🔑 Tips: Selalu pastikan untuk memilih mirror yang resmi dari proyek Slackware atau dikelola oleh komunitas institusi akademik terpercaya agar integritas paket biner yang Anda pasang tetap aman dari manipulasi pihak ketiga.

2. NixOS (nix.conf mirror)

Bergeser ke ekosistem yang sepenuhnya berbeda, NixOS mengadopsi pendekatan manajemen paket fungsional deklaratif melalui sistem paket bernama nix. Tidak seperti distro konvensional yang mengunduh kode sumber mentah atau arsip biner biasa, Nix mengandalkan skema penyerahan biner siap pakai dari server yang disebut dengan substituters.

Kecepatan instalasi NixOS akan meningkat drastis apabila kita menambahkan cermin substituter lokal yang responsif. Konfigurasi ini dapat diatur dengan dua cara alternatif: mengekstrak langsung pada berkas tingkat rendah /etc/nix/nix.conf atau mengintegrasikannya secara permanen di dalam berkas arsitektur sistem utama Anda, yaitu configuration.nix.

# Edit file konfigurasi nix.conf
sudo nano /etc/nix/nix.conf
# Tambahkan mirror substituter (contoh: cache.nixos.org dan mirror lokal)
substituters = https://cache.nixos.org https://mirror.sjtu.edu.cn/nix-channels/store
trusted-public-keys = cache.nixos.org-1:xxxxxxxxxxxxxx mirror.sjtu.edu.cn-1:yyyyyyyyyyyyyy
# Simpan lalu jalankan
sudo nix-channel --update
nix-env -u

🌍 Untuk daftar informasi server mirror resmi serta opsi node substituter regional NixOS lainnya secara global, Anda dapat langsung mengunjungi dokumentasi resminya di tautan nixos.org/mirrors.

Kesimpulan

Mengonfigurasi jalur distribusi paket yang optimal terbukti krusial dalam memangkas waktu tunggu pemeliharaan server maupun mesin kerja harian kita. Dua metode yang telah kita bahas merangkum perbedaan filosofi mendasar antar kedua distro:

  • Slackware → Menuntut kita mengedit berkas statis /etc/slackpkg/mirrors secara manual, diikuti dengan pembaruan tanda tangan GPG via utilitas slackpkg.
  • NixOS → Memanfaatkan arsitektur berbasis token pada berkas konfigurasi /etc/nix/nix.conf untuk mendefinisikan URL substituter eksternal tepercaya baru.

Dengan memetakan mirror repositori yang tepat dan memiliki latensi rendah, baik ekosistem klasik Slackware maupun ekosistem modern NixOS Anda akan mampu melakukan proses pembaruan, kompilasi, serta instalasi paket secara jauh lebih gesit dan ngebut 🚀.
