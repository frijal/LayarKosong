🤔 Kenapa Banyak Branch di Repo GitHub?

Kalau kamu buka repo GitHub dan lihat branch-nya bejibun, jangan panik dulu. Banyak branch itu wajar banget, apalagi kalau repo dipakai rame-rame. Yuk kita bahas alasan utamanya:

🔹 Alasan Kenapa Branch Bisa Banyak

  • Kolaborasi tim: setiap orang bikin branch sendiri buat fitur/bugfix biar tidak ganggu main.
  • Eksperimen: kadang ada branch khusus buat coba-coba ide baru.
  • Workflow tertentu: misalnya Git Flow, yang memang butuh banyak branch (develop, release, hotfix, dll).
  • Branch lama belum dihapus: setelah merge, branch kadang lupa dihapus, jadi numpuk.

🔹 Apa yang Bisa Dilakukan?

Kalau repo kamu penuh branch, coba:

  1. Hapus branch yang sudah di-merge dan tidak dipakai lagi.
  2. Pakai branch protection biar branch utama tetap aman.
  3. Gunakan penamaan branch yang jelas, misalnya feature/login atau bugfix/issue-123.
