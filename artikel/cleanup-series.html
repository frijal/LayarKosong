🪄 Cleanup Series

Seri artikel ini menjelaskan bagaimana cara menjaga proyek GitHub Pages tetap bersih, ringan, dan efisien dengan melakukan pembersihan dependensi Node.js menggunakan dua pendekatan: interaktif dan otomatis penuh. Cocok untuk kamu yang sering mengembangkan situs statis dengan workflow berbasis package.json.

🧹 cleanup.js (Interaktif)

Versi interaktif dengan inquirer yang memungkinkan kamu memilih paket mana yang ingin dihapus. Cocok untuk penggunaan lokal saat kamu ingin kontrol penuh atas dependensi.

Skrip non-interaktif yang langsung menghapus semua paket tak terpakai dan reinstall dependensi. Ideal untuk dijalankan di GitHub Actions tanpa campur tangan pengguna.

Dengan dua alat ini, kamu bisa memastikan proyek Node.js untuk GitHub Pages tetap bersih dan cepat di-build. Gunakan versi interaktif saat bekerja secara lokal, dan versi dapat di CI/CD agar workflow berjalan mulus tanpa repot konfigurasi ulang.
