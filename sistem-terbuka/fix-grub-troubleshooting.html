Troubleshooting GRUB Bootloader

Terkadang, setelah menjalankan `update-grub`, sistem masih gagal booting. Berikut adalah beberapa langkah tambahan untuk mengatasi masalah pada GRUB di Debian/Ubuntu.

1. Perbaikan untuk Sistem UEFI

Jika sistem Anda menggunakan UEFI, pastikan partisi EFI (biasanya `/dev/sda1`) ter-mount dengan benar di dalam `chroot` sebelum menginstal GRUB.

# Mount partisi EFI Anda
sudo mount /dev/sda1 /mnt/boot/efi
# Instal ulang GRUB untuk UEFI
grub-install --target=x86_64-efi --efi-directory=/boot/efi
update-grub

2. Gunakan Boot-Repair (Alternatif Mudah)

Jika perbaikan manual terasa rumit, utilitas **Boot-Repair** dapat mendeteksi dan memperbaiki sebagian besar masalah GRUB secara otomatis. Jalankan dari Live USB Ubuntu.

sudo add-apt-repository ppa:yannubuntu/boot-repair
sudo apt update
sudo apt install -y boot-repair && boot-repair

3. Cek Mode Boot & File Sistem

Pastikan mode boot di BIOS/UEFI Anda (Legacy vs UEFI) konsisten dengan cara OS diinstal. Jika GRUB masih gagal membaca kernel, periksa integritas file sistem dengan `fsck`.

# Ganti sda1 dengan partisi root Anda
sudo fsck -f /dev/sda1
