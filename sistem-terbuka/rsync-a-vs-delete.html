Bedanya `rsync -a` vs `rsync -a --delete`

Penjelasan Santai Kapan Harus Pakai (dan Kapan NGGAK)

Jawabannya simpel, Bro: **tergantung kebutuhan!**

Parameter `-a` (archive) itu emang sakti, tapi dia **NGGAK** otomatis menyertakan `--delete`.

`rsync -a` itu sebenernya singkatan keren untuk `-rlptgoD`. Gampangnya, dia bakal:

  • Nyalin file & folder secara rekursif (nurun ke semua sub-folder).
  • Jaga *permissions* (izin file).
  • Jaga *timestamps* (waktu file dibuat/diubah).
  • Jaga *symbolic links*.
  • Jaga *owner* dan *group* file (kalau kamu jalankan sebagai root).

Tapi, dia *nggak* akan menghapus apa-apa di tujuan. Nah, `--delete` ini adalah "bumbu" tambahan yang fungsinya spesifik banget.

Skenario Penggunaan `rsync`

Biar gampang, kita bagi jadi dua skenario utama:

üîÑ Skenario 1: Pakai `rsync -a --delete` (Sinkronisasi / Mirroring)

Kamu **HARUS** pakai `--delete` kalau kamu mau folder destinasi (tujuan) jadi **cermin (mirror) yang 100% identik** dengan folder sumber (source).

Apa yang terjadi:

  • File baru di sumber ‚ûî Disalin ke destinasi.
  • File yang berubah di sumber ‚ûî Diperbarui di destinasi.
  • File yang dihapus di sumber ‚ûî **Ikut Dihapus di destinasi.**

Contoh kasus:

Menyinkronkan folder kerja kamu ke hard drive eksternal. Kamu mau apa yang ada di eksternal itu *selalu sama persis* dengan apa yang ada di laptop. Kalau kamu hapus file sampah di laptop, kamu juga mau file sampah itu kehapus di backup eksternal biar nggak menuh-menuhin.

# Perintah untuk mirroring
rsync -a --delete /folder/sumber/ /folder/tujuan/

üì• Skenario 2: Pakai `rsync -a` (Tanpa `--delete`)

Kamu **NGGAK PERLU** pakai `--delete` kalau tujuan kamu cuma mau **menambah atau memperbarui file** di destinasi, tanpa menghapus apa pun.

Apa yang terjadi:

  • File baru di sumber ‚ûî Disalin ke destinasi.
  • File yang berubah di sumber ‚ûî Diperbarui di destinasi.
  • File yang dihapus di sumber ‚ûî **Tetap AMAN di destinasi.**

Contoh kasus:

Membuat backup "aditif" (bertambah terus). Misalnya kamu punya folder "Foto Keren" di laptop. Kamu backup ke server. Suatu hari kamu nggak sengaja hapus foto di laptop. Kalau kamu pakai `rsync -a` (tanpa `--delete`), foto yang udah telanjur ke-backup di server *nggak akan ikut kehapus*. Server kamu jadi kayak "arsip abadi" dari semua file yang pernah ada.

# Perintah untuk backup (hanya nambah/update)
rsync -a /folder/sumber/ /folder/tujuan/

Contoh di Terminal

Ini dia contoh tampilan `rsync` di terminal, mirip dengan yang kamu kirim. Ini ngebantu banget buat liat file apa aja yang ditransfer.

**‚ö†Ô∏è PERINGATAN KERAS!**

Selalu, selalu, dan **selalu hati-hati** pas pakai `--delete`. Kalau kamu salah ketik path (misalnya kebalik antara sumber dan tujuan), kamu bisa secara nggak sengaja menghapus semua data penting kamu di destinasi!

Sebelum beneran eksekusi, biasakan pakai `--dry-run` (atau `-n`) dulu. Perintah ini cuma simulasi, nunjukin apa yang *akan* `rsync` lakukan tanpa benar-benar melakukannya.

# Cuma simulasi (dry run), tambahin -v (verbose) biar detail
rsync -avn --delete /folder/sumber/ /folder/tujuan/
