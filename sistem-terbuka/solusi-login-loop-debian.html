Solusi Gagal Login (Login Loop) di Debian Setelah Install

Ini masalah yang cukup sering terjadi di dunia Linux, biasa disebut "login loop". Kamu sudah yakin username dan password benar, tapi setiap kali login malah mental kembali ke halaman login. Tenang, ini bukan berarti instalasi Debian kamu gagal total dan biasanya bisa diatasi.

Langkah Wajib Pertama: Masuk ke Terminal TTY

Karena tidak bisa masuk ke desktop, kita akan perbaiki lewat "pintu belakang", yaitu terminal TTY. Di halaman login, tekan tombol:

Ctrl + Alt + F3 (atau F4, F5, F6)

Layar akan berubah menjadi hitam dengan teks (command-line). Login di sini dengan username dan password kamu. Jika berhasil, kamu siap untuk lanjut ke langkah berikutnya.

Cara Mengatasi Login Loop (Coba dari No. 1)

1. Perbaiki Driver Kartu Grafis (Penyebab Paling Umum)

Ini adalah biang keladi 90% kasus login loop, terutama jika kamu pakai kartu grafis NVIDIA.

A. Cek Kartu Grafis

lspci | grep -i vga

B. Solusi untuk NVIDIA

Kita perlu install driver proprietary. Pertama, izinkan Debian mengambil software non-free dengan mengedit file sources.list: 

sudo nano /etc/apt/sources.list

Tambahkan contrib non-free non-free-firmware di setiap baris sumber. Contoh: 

deb http://deb.debian.org/debian/ bookworm main contrib non-free non-free-firmware

Tekan Ctrl+X, lalu Y, dan Enter untuk menyimpan. Lanjutkan dengan update dan instalasi: 

sudo apt update
sudo apt install nvidia-driver

Terakhir, reboot komputer:

sudo reboot

C. Solusi untuk AMD/Intel

Biasanya sudah aman, tapi coba lakukan update penuh untuk jaga-jaga: 

sudo apt update
sudo apt full-upgrade
sudo reboot

2. Perbaiki Hak Akses (Permission) di Home Directory

Kadang, sistem tidak bisa membuat file sesi login karena permission folder home kamu salah. Perbaiki dengan perintah ini di TTY: 

sudo chown -R $(whoami):$(whoami) $HOME
sudo chown $(whoami):$(whoami) ~/.Xauthority

Setelah itu, coba sudo reboot.

3. Cek Sisa Ruang Disk

Jika partisi root (/) atau /home penuh 100%, kamu juga tidak akan bisa login. Cek dengan: 

df -h

Kalau ada yang 100%, bersihkan cache untuk dapat sedikit ruang:

sudo apt clean

4. Install Ulang Desktop Environment

Jika semua cara gagal, mungkin ada file desktop yang rusak. Install ulang bisa jadi solusi.

  • Untuk GNOME (default):
    • sudo apt install --reinstall gnome-shell gdm3
  • Untuk KDE Plasma:
    • sudo apt install --reinstall plasma-desktop sddm
  • Untuk Xfce:
    • sudo apt install --reinstall xfce4 lightdm

Pilih sesuai desktop yang kamu install, lalu sudo reboot.
