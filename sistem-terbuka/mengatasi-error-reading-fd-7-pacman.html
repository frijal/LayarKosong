Cara Ampuh Mengatasi Error reading fd 7 di Pacman

Ditulis dari pengalaman lapangan praktisi Linux | Update terbaru untuk pengguna Arch & Turunannya

Buat kamu yang hobi banget pakai distro Arch Linux atau saudara-saudaranya kayak Manjaro dan EndeavourOS, rutinitas bersihin cache udah pasti jadi makanan sehari-hari. Tapi, gimana rasanya kalau lagi asyik ngejalanin perintah pembersihan, tiba-tiba terminal kamu neriakin pesan error kayak gini?

Cache directory: /var/cache/pacman/pkg/ :: Do you want to remove all other packages from cache? [Y/n] removing old packages from cache... error: could not open file /var/cache/pacman/pkg/download-: Error reading fd 7

Bikin panik? Tidak perlu! 😌 Sebagai sesama pengguna Linux yang udah sering kena jebakan sistem, mari kita kupas masalah ini santai aja pakai metode 5W+1H supaya pemahaman kamu makin mantap, tidak cuma asal copy-paste kode aja.

Apa (What) Sebenarnya Error Ini?

Pesan error ini terjadi karena alat manajemen paket kita, si Pacman, mencoba menganggap direktori download sementara (contohnya /var/cache/pacman/pkg/download-XXXXXX) sebagai sebuah file biasa saat dia lagi sibuk melakukan proses bersih-bersih cache. Padahal itu kan folder, bukan file paket, makanya sistem bingung dan protes menolak membacanya.

Siapa (Who) yang Biasanya Kena Masalah Ini?

Ini adalah "penyakit" yang sangat lumrah menimpa para pengguna keluarga besar Arch Linux. Terutama buat kamu yang sudah melakukan update sistem dan menggunakan pacman versi 7.0 ke atas.

Di Mana (Where) Error Ini Bersarang?

Sumber masalahnya ada di dalam sistem operasi kamu, lebih spesifiknya nongkrong di direktori cache bawaan, yaitu di jalur /var/cache/pacman/pkg/.

Kapan (When) Masalah Ini Muncul?

Error "Error reading fd 7" ini biasanya baru muncul ke permukaan ketika kamu mengeksekusi perintah untuk membersihkan cache paket (seperti pacman -Sc). Kamu cenderung tidak sadar ada file nyangkut sampai kamu mencoba bersih-bersih sampah sistem.

Kenapa (Why) Hal Ini Bisa Terjadi?

Ada penjelasan teknis yang lumayan menarik di balik ini:

  • Bug Sandboxing: Mulai dari pacman versi 7.0+, developer mulai menerapkan sistem sandboxing baru. Saat mengunduh paket, pacman membuat direktori-direktori sementara (temporary directories) untuk menampung unduhan tersebut.
  • Gagal Hapus (Failed Deletion): Nah, kalau koneksi internet kamu mendadak putus pas lagi update, atau kamu iseng mencet Ctrl+C buat ngebatalin update di tengah jalan, pacman seringkali gagal membersihkan folder sementara ini secara otomatis. Sisa-sisa folder kosong atau setengah matang inilah yang tertinggal dan membuat error.

Bagaimana (How) Cara Ampuh Memperbaikinya?

Karena folder-folder yang error ini sejatinya hanyalah direktori kosong sementara atau sisa-sisa dari proses unduhan yang terputus, kamu bisa menyingkirkannya secara manual dengan sangat aman tanpa ngerusak sistem utama kamu. 🚀

Langkah 1: Hapus Direktori Nyasar Secara Manual

Jalankan perintah berikut di terminal kamu untuk menghapus HANYA direktori sementara yang bikin masalah:

sudo rm -rf /var/cache/pacman/pkg/download-*

Tips: Pastikan kamu mengetiknya dengan benar ya, bintang () di akhir itu penting supaya cuma folder berawalan "download-" yang tersapu bersih.

Langkah 2: Coba Bersihkan Cache Lagi

Setelah file sampahnya hilang, coba jalankan ulang perintah pembersihan andalan kamu. Sekarang harusnya udah mulus tanpa hambatan:

sudo pacman -Sc

💡 Alternatif Super Praktis: Gunakan Paccache

Buat kamu yang pengen manajemen cache lebih elegan dan teratur, aku sangat menyarankan pakai skrip paccache (ini adalah bagian dari paket pacman-contrib, pastikan kamu udah install ya). Cara ini lebih aman dari sekadar menghapus semua cache pakai opsi -Sc.

Menghapus semua file cache, tapi menyisakan 3 versi terbaru (buat jaga-jaga kalau perlu downgrade):

sudo paccache -r

Menghapus semua versi dari paket yang sudah kamu uninstall dari sistem:

sudo paccache -ruk0
