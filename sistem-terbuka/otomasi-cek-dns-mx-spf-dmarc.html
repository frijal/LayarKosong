Otomasi Cek Propagasi DNS: MX, SPF, dan DMARC untuk dalam.web.id

5 Maret 2026 | Fakhrul Rijal

Opsi 1: Pakai Bun (Modern & Ngebut)

Kalau kamu sudah pakai **Bun** sepenuhnya, script ini jauh lebih rapi dan bisa memberikan output yang terstruktur (JSON atau teks berwarna).

// dns-check.ts
import { resolveMx, resolveTxt } from "node:dns/promises";

const domain = "dalam.web.id";

async function checkDNS() {
  console.log("🔍 Mengecek DNS untuk " + domain + "...");

  try {
    const mx = await resolveMx(domain);
    console.log("✅ MX Records:", mx);

    const txt = await resolveTxt(domain);
    const spf = txt.find(t => t[0].includes("v=spf1"));
    console.log(spf ? "✅ SPF: " + spf : "❌ SPF tidak ditemukan");

    const dmarc = await resolveTxt("_dmarc." + domain);
    console.log(dmarc ? "✅ DMARC: " + dmarc : "❌ DMARC tidak ditemukan");
  } catch (e) {
    console.error("⚠️ Terjadi kesalahan atau record belum siap.");
  }
}

checkDNS();

Opsi 2: Pakai Bash (Klasik & Universal)

Script ini cocok kalau kamu lagi malas *install* runtime dan pengen yang langsung jalan di Arch Linux kesayangan kamu.

#!/bin/bash
# dns-check.sh
DOMAIN="dalam.web.id"

echo "Checking DNS Records for $DOMAIN..."
echo "------------------------------------"

echo "📦 MX Records:"
dig +short MX $DOMAIN

echo -e "\n🛡️ SPF Record:"
dig +short TXT $DOMAIN | grep "v=spf1"

echo -e "\n🔐 DMARC Record:"
dig +short TXT _dmarc.$DOMAIN | grep "v=DMARC1"

echo "------------------------------------"
echo "Done!"

Apa Itu Otomasi Cek DNS? (5W+1H)

**What:** Otomasi ini adalah penggunaan script sederhana berbasis Bun atau Bash untuk melakukan *query* DNS secara berkala guna memastikan record MX, SPF, dan DMARC sudah menyebar dengan benar di internet.

**Who:** Ditujukan untuk pemilik domain `dalam.web.id` (kamu sendiri!) atau developer yang baru saja melakukan migrasi email ke Cloudflare.

**Why:** Karena mengecek satu per satu lewat terminal (`dig` atau `nslookup`) secara manual itu melelahkan dan rawan salah ketik. Kita butuh cara yang "sekali klik langsung beres".

**Where:** Bisa dijalankan di komputer lokal (Linux/macOS), server, atau bahkan sebagai bagian dari GitHub Actions.

**When:** Paling krusial dijalankan sesaat setelah kamu mengubah *setting* DNS di dashboard Cloudflare sampai statusnya terverifikasi.

**How:** Dengan menggunakan library DNS bawaan dari Bun atau perintah `dig` pada Bash yang dibungkus dalam perulangan atau script terencana.

Prinsip E-E-A-T dalam Konfigurasi DNS

Data yang akurat adalah segalanya. Dengan mengotomasi pengecekan ini, kamu menunjukkan **Expertise** dan **Authoritativeness** dalam pengelolaan sistem. Hal ini juga memastikan **Trustworthiness** karena email yang kamu kirim dari domain komunitas tidak akan dianggap spam oleh server tujuan berkat record SPF dan DMARC yang valid.
