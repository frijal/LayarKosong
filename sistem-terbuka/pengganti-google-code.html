Apa Pengganti Google Code Sekarang? (Dulu Komunitas, Kini Bisnis)

Ini adalah kelanjutan alami dari cerita penutupan Google Code. Jadi, setelah ditutup, apa yang menggantikannya? Dan apakah Google masih punya layanan serupa? Jawabannya: "ya dan tidak".

1. Apa yang Menggantikan Fungsi Google Code?

Secara global, fungsi Google Code sebagai "rumah" bagi komunitas *open-source*tidak digantikan oleh Google, melainkan oleh platform lain yang sudah ada dan terus berkembang.

Pemenang terbesarnya jelas adalah GitHub.

Komunitas developer secara massal sudah "jatuh cinta" dan pindah ke GitHub. Alasannya? Alur kerja "social coding" mereka (Fork, Pull Request) terbukti jauh lebih modern, cepat, dan asyik untuk Obsesi. Layanan lain seperti GitLab dan Bitbucket juga mengambil bagian yang signifikan di pasar.

Jadi, yang menggantikan Google Code adalah evolusi pasar. GitHub secara alami menjadi standar de facto (standar yang diterima semua orang) untuk proyek *open-source*.

Google pun menyadari ini. Mereka tidak melawan arus. Google bahkan memindahkan sebagian besar proyek open-source publik raksasa mereka (seperti TensorFlow, Go, dan Kubernetes) ke GitHub. Tujuannya? Tentu saja agar lebih dekat dengan komunitas developer yang sekarang semuanya ada di sana.

2. Apakah Google Masih Punya Layanan Serupa?

Ya, Google masih memiliki layanan hosting Git, tapi tujuannya sangat berbeda. Layanan ini bukan untuk menyaingi GitHub sebagai platform komunitas.

Layanan mereka yang sekarang bernama Google Cloud Source Repositories.

Ini adalah bagian dari Google Cloud Platform (GCP). Perbedaan utamanya sangat jelas, dan ini menunjukkan pergeseran strategi Google:

Google Code (LAMA)

  • Target: Publik / Komunitas Open-Source.
  • Tujuan: Menumbuhkan ekosistem open-source.
  • Biaya: Gratis.
  • Status: Sudah Ditutup (2015).

Cloud Source Repositories (BARU)

  • Target: Pelanggan Bisnis / Enterprise.
  • Tujuan: Menyimpan kode aplikasi privat dengan aman.
  • Biaya: Layanan Berbayar (bagian dari GCP).
  • Status: Aktif dan dikembangkan.

Kesimpulan Singkat

Google sudah "lempar handuk" untuk bersaing dengan GitHub dalam urusan komunitas open-source publik.

Sebagai gantinya, mereka sekarang fokus menyediakan layanan hosting Git privat yang canggih untuk pelanggan bisnis mereka, yang terintegrasi penuh dengan ekosistem Google Cloud (seperti Cloud Build untuk CI/CD, App Engine, dll).

Jadi, Google Code digantikan oleh GitHub (untuk komunitas), dan Google membuat Cloud Source Repositories (untuk bisnis).
