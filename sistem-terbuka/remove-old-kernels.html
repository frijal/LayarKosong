Hapus Kernel Lama di Linux: Simpan yang Penting, Bersihkan yang Aman

Kernel lama yang menumpuk bisa dibersihkan untuk menghemat ruang disk. Namun, penting untuk melakukannya dengan aman. Panduan ini menunjukkan cara praktis untuk mengelola kernel di sistem berbasis Debian/Ubuntu.

Langkah-langkah Membersihkan Kernel

1. Lihat Daftar Kernel yang Terpasang

Gunakan perintah dpkg untuk melihat semua paket kernel (image dan headers) yang ada di sistem Anda.

dpkg -l "linux-image*" "linux-headers*"

Perhatikan outputnya. Baris yang diawali dengan "ii" menandakan paket terpasang, sedangkan "rc" berarti sudah dihapus tapi konfigurasinya masih ada.

2. Hapus Kernel Secara Spesifik (Opsional)

Jika Anda tahu persis versi mana yang ingin dihapus, gunakan apt purge. Perintah ini akan menghapus paket beserta file konfigurasinya.

sudo apt purge linux-image-6.1.0-10-amd64

3. Gunakan Cara Otomatis (Direkomendasikan)

Cara yang lebih aman dan mudah adalah menggunakan apt autoremove. Perintah ini secara cerdas akan menghapus semua kernel lama, kecuali dua versi terbaru (kernel yang sedang berjalan dan satu versi sebelumnya sebagai cadangan).

sudo apt autoremove --purge

Opsi --purge memastikan file konfigurasi juga ikut dibersihkan.

Tips Terbaik: Biasakan menyimpan minimal dua kernel terakhir. Jika kernel terbaru menyebabkan masalah (misalnya, WiFi tidak berfungsi), Anda bisa boot menggunakan kernel sebelumnya dari menu GRUB.
