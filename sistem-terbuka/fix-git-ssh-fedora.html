Kalau kamu pakai Fedora, RHEL, CentOS, atau turunannya, terus nemu pesan error ini saat `git fetch`:

error: cannot run ssh: No such file or directory
fatal: unable to fork

Tenang aja ‚Äî ini error umum yang muncul karena **Git nggak nemuin executable SSH** di sistem kamu. Biasanya karena paket `openssh-clients` belum terinstal.

  • **OpenSSH belum diinstal** ‚Äî Fedora dan RHEL memisahkan paket server dan client.
  • **PATH tidak mengenali lokasi ssh** ‚Äî misal kamu jalankan dari chroot, container minimal, atau sistem custom.

Untuk Fedora dan RHEL 8+, gunakan perintah `dnf` berikut:

Untuk versi lama CentOS atau RHEL 7, pakai:

sudo yum install -y openssh-clients

# fix-git-ssh-fedora.sh
set -e
echo "üîç Mengecek apakah openssh-clients sudah terinstal..."

  if command -v dnf >/dev/null 2>&1; then
    echo "üì¶ Menginstal openssh-clients dengan dnf..."
    sudo dnf install -y openssh-clients
  else
    echo "üì¶ Menginstal openssh-clients dengan yum..."
    sudo yum install -y openssh-clients
  fi

  echo "‚úÖ openssh-clients sudah terinstal."
fi
if ! command -v ssh >/dev/null; then
  echo "‚ùå ssh tidak ditemukan di PATH."
  echo "PATH: $PATH"
  exit 1
fi

echo "‚úÖ SSH siap digunakan untuk Git!"

Kemudian pastikan binary `ssh` sudah terdeteksi:
