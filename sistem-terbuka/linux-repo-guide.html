Optimasi Update & Instalasi Paket untuk Pejuang Open Source Indonesia

Halo teman-teman di **Layar Kosong**! Pernah nggak sih lagi asyik mau install `neofetch` atau update sistem, eh malah dapet kecepatan "siput" cuma beberapa KBps? Padahal internet di Balikpapan lagi kenceng-kencengnya.

Nah, masalahnya biasanya bukan di ISP kamu, tapi di **Mirror Repository** yang kamu pake. Secara default, Linux seringkali nge-set mirror global (US atau Eropa). Di artikel ini, saya bakal kasih tau rahasia milih server terdekat biar `sudo apt update` kamu secepat kilat!

Keluarga Debian & Ubuntu

Keluarga ini adalah yang paling populer. Mereka pakai manajer paket **APT**. Masalahnya, mirror default sering kena limit bandwidth kalau lagi jam sibuk.

**ðŸ’¡ Tips Balikpapan:** Pilihlah mirror lokal Indonesia seperti milik *DataUtama*, *Kambing UI*, atau *Poliwangi*.

Untuk Ubuntu, kamu bisa pakai cara keren lewat terminal untuk otomatis cari yang tercepat:

# Cari mirror tercepat di Indonesia
sudo sed -i 's|http://archive.ubuntu.com/ubuntu/|mirror://mirrors.ubuntu.com/mirrors.txt|g' /etc/apt/sources.list

Arch Linux & Manjaro

Arch Linux itu "Rolling Release", jadi update-nya sering banget dan gede-gede. Kalau mirror kamu lemot, bisa emosi jiwa nungguinnya.

Gunakan tool sakti bernama `reflector`. Tool ini bakal ngetes ping ke semua mirror dan milihin yang paling segar buat kamu.

# Ambil 10 mirror Indonesia/Singapore tercepat & urutkan berdasarkan speed
sudo reflector --country Indonesia,Singapore --latest 10 --sort rate --save /etc/pacman.d/mirrorlist

Fedora & NixOS

Fedora pakai **DNF** yang sebenarnya sudah punya fitur `fastestmirror`. Kamu cuma perlu aktifin di konfigurasinya.

Buka file `/etc/dnf/dnf.conf` dan tambahkan baris: `fastestmirror=True`. Dijamin kerasa bedanya!

Sedangkan untuk **NixOS**, karena sistemnya deklaratif, kamu tinggal atur di `configuration.nix` kamu untuk mengarah ke substituter terdekat.

Kesimpulan Akhir

Memilih repository itu ibarat milih jalan tol. Kalau ada yang dekat dan kosong, ngapain lewat yang jauh dan macet? Jangan lupa untuk selalu melakukan `sync` ulang setelah ganti mirror ya!
