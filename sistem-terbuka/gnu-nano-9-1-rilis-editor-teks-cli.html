GNU nano 9.1 Rilis: Editor Teks CLI Populer dengan Perbaikan dan Fitur Baru

Setelah dua setengah bulan sejak rilis besar 9.0, GNU nano 9.1 hadir sebagai pembaruan kecil namun penting bagi para pengguna setia editor teks berbasis terminal ini.

GNU nano adalah salah satu editor teks command-line yang paling banyak digunakan di dunia Linux dan Unix. Kehadirannya yang ringan, mudah digunakan, dan tersedia secara default di hampir semua distribusi membuatnya menjadi andalan bagi administrator sistem, pengembang, dan pengguna biasa yang mengedit file konfigurasi atau skrip dari terminal.

Rilis terbaru, GNU nano 9.1, diumumkan hari ini oleh tim pengembang. Meskipun tergolong point release, versi ini membawa sejumlah perubahan yang meningkatkan fleksibilitas dan stabilitas editor. Bagi Anda yang sehari-hari berinteraksi dengan nano, pembaruan ini layak untuk diketahui.

Gambar di atas menampilkan antarmuka GNU nano 9.1 yang masih setia dengan tampilan minimalis khasnya. Namun di balik kesederhanaan itu, terdapat peningkatan fungsionalitas yang membuat pengeditan teks semakin nyaman.

Apa yang Baru di GNU nano 9.1

Rilis ini memang tidak sebesar versi 9.0, tetapi beberapa perubahan cukup berarti bagi pengguna power. Berikut rincian fitur-fitur baru yang diperkenalkan.

Key Binding M-Ins dan M-Del Kini Rebindable

Sebelumnya, pintasan keyboard M-Ins (Alt+Insert) dan M-Del (Alt+Delete) memiliki fungsi tetap yang tidak dapat diubah. Kini, pengguna dapat mengikat ulang kedua kombinasi tombol tersebut sesuai kebiasaan mereka. Ini memberikan fleksibilitas lebih bagi mereka yang telah terbiasa dengan tata letak keyboard tertentu atau ingin menyelaraskan dengan editor lain.

Pergeseran Viewport yang Lebih Cerdas

Saat melakukan pencarian, nano 9.1 akan menggeser viewport ke kiri jika memungkinkan, sehingga hasil pencarian tidak tersembunyi di tepi layar. Perubahan kecil ini terlihat sepele, tetapi sangat membantu ketika bekerja dengan baris teks yang panjang. Pengguna tidak perlu lagi menggulir manual untuk melihat konteks hasil pencarian.

Penghapusan Dukungan Format Mac Lama

Dukungan untuk membaca dan menulis file dalam format Mac klasik (dengan carriage return tunggal) dihapus. Format ini sudah usang dan jarang ditemui, sehingga penghapusannya menyederhanakan kode dan mengurangi potensi kebingungan saat membuka file dari sistem yang sangat lawas.

Deprecasi Toggle ^T

Tombol pintas ^T yang sebelumnya berfungsi untuk beralih antara WhereIs (pencarian) dan GotoLine (lompat ke baris) kini dianggap usang. Pengguna disarankan untuk menggunakan pintasan terpisah yang lebih intuitif. Langkah ini merupakan bagian dari upaya membersihkan antarmuka dan mengurangi ambiguitas fungsi.

Perbaikan Bug Penting

Selain fitur baru, GNU nano 9.1 juga memperbaiki dua masalah yang cukup mengganggu bagi pengguna yang mengandalkan fitur backup dan penanganan file saat terjadi crash.

Backup Gagal dan Timestamp Salah

Jika opsi --backup diaktifkan, sebelumnya nano dapat gagal membuat file cadangan atau memberikan timestamp yang salah. Hal ini tentu merugikan karena backup adalah jaring pengaman penting saat mengedit file sistem. Kini, proses backup berjalan dengan benar dan timestamp sesuai dengan waktu sebenarnya.

Izin File .save Tidak Tepat

Ketika nano mati mendadak atau crash, ia menyimpan keadaan ke file dengan ekstensi .save. Namun, file tersebut sebelumnya tidak mewarisi izin dan kepemilikan dari file asli, yang bisa menyebabkan masalah keamanan atau akses. Perbaikan ini memastikan bahwa file .save memiliki izin dan kepemilikan yang sama dengan file yang sedang diedit, sehingga proses pemulihan menjadi lebih aman.

Mengenal Rilis Sebelumnya: GNU nano 9.0

Untuk memahami konteks pembaruan 9.1, ada baiknya kita mengenang apa yang dibawa oleh GNU nano 9.0 pada bulan April 2026. Rilis besar itu memperkenalkan beberapa fitur yang mengubah cara pengguna melihat dan menavigasi teks.

Scroll Horizontal Bertahap

Versi 9.0 menambahkan kemampuan menggulir viewport secara horizontal dalam langkah-langkah sebesar satu tabsize. Ini sangat membantu saat mengedit baris yang sangat panjang, karena pergerakan menjadi lebih halus dan terkendali.

Scroll Semua Baris Secara Serempak

Ketika kursor hampir keluar dari layar, semua baris akan bergeser bersama secara horizontal. Ini memberikan pengalaman yang lebih alami dibandingkan sebelumnya yang hanya menggeser baris tempat kursor berada.

Key Binding Tambahan

Pintasan M-Left, M-Right, M-Up, dan M-Down kini dapat diikat ulang sesuai keinginan pengguna, memberikan kontrol lebih besar atas navigasi.

Pembatalan Rekaman Makro

Jika pengguna membatalkan rekaman makro segera setelah memulainya, makro yang ada sebelumnya tidak akan hilang. Ini menghindari kehilangan makro berharga akibat kesalahan penekanan tombol.

Navigasi dengan Scrollbar

Dengan mengaktifkan opsi --mouse dan --indicator, pengguna dapat mengklik area scrollbar untuk melompat ke bagian buffer secara kasar. Fitur ini sangat memudahkan navigasi pada file besar.

GNU nano 9.0 menjadi fondasi yang kuat, dan 9.1 adalah penyempurnaan dari fondasi tersebut.

Cara Mendapatkan GNU nano 9.1

Bagi Anda yang ingin segera mencoba versi terbaru, tersedia dua jalur utama: kompilasi dari kode sumber atau menunggu paket dari distribusi Linux kesayangan Anda.

Kompilasi dari Sumber

Jika Anda terbiasa mengkompilasi sendiri, Anda dapat mengunduh tarball sumber dari situs resmi GNU nano. Proses kompilasi cukup sederhana dengan perintah ./configure && make && sudo make install. Pastikan Anda telah menginstal pustaka pengembangan yang diperlukan.

Melalui Manajer Paket Distribusi

Untuk sebagian besar pengguna, cara paling mudah adalah menunggu pembaruan masuk ke repositori stabil distribusi Anda. Distribusi seperti Arch Linux, Fedora, Ubuntu, dan Debian biasanya akan menyertakan rilis baru dalam hitungan hari atau minggu. Anda dapat memeriksa dengan perintah manajer paket masing-masing.

Bagi yang menggunakan snap atau flatpak, kemungkinan versi ini juga akan segera tersedia di kanal-kanal tersebut.

Mengapa GNU nano Tetap Relevan?

Di tengah maraknya editor modern seperti VS Code, Neovim, dan Emacs, GNU nano justru mempertahankan posisinya sebagai editor yang straightforward dan selalu tersedia. Ketika Anda harus mengedit file di server tanpa antarmuka grafis, nano adalah salah satu opsi yang paling mudah diingat dan digunakan.

Rilis 9.1 menunjukkan bahwa proyek ini terus dipelihara dengan perhatian terhadap detail dan kebutuhan pengguna. Pembaruan kecil seperti perbaikan backup dan fleksibilitas key binding mungkin tidak mengubah cara kerja secara drastis, tetapi meningkatkan keandalan dan kenyamanan sehari-hari.

Bagi para pengguna setia, pembaruan ini adalah alasan untuk tetap menggunakan nano. Bagi yang baru mengenal, ini adalah waktu yang tepat untuk mulai mencoba dan merasakan kemudahan editor teks yang sudah berusia lebih dari dua dekade ini.
