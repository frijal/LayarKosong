Sumber: Infografis Linux File System (2025)

Linux itu rapi banget ngatur filenya. Bayangin kayak rumah besar ‚Äî semua punya tempatnya masing-masing!
/bin
‚ÄúTempatnya perintah-perintah dasar yang wajib ada.‚Äù

Isinya: `ls`, `cat`, `rm`, `cp`, `mv`, `echo`, dll.

Semua orang bisa pakai, bahkan saat sistem lagi ‚Äúrusak parah‚Äù.

Kalau ini ilang? Ya... Linux bakal nangis. üò≠

Contoh: /bin/ls, /bin/bash, /bin/mkdir
/boot
‚ÄúRumahnya file-file buat nyalain Linux.‚Äù

Ada: `vmlinuz` (kernel), `initramfs`, `grub.cfg`.

GRUB baca dari sini buat nampilin menu boot.

Jangan sembarangan edit, bisa-bisa laptop nggak nyala!

ls /boot ‚Üí vmlinuz-6.5.0-14-generic
/dev
‚ÄúSemua hardware di sini jadi 'file'.‚Äù

Contoh: `/dev/sda` = harddisk pertama, `/dev/null` = lubang hitam, `/dev/random` = angka acak.

Keren, kan? Linux bilang: **‚ÄúSemua adalah file!‚Äù**

ls /dev ‚Üí sda, null, random, tty, usb
/etc
‚ÄúPusat pengaturan sistem.‚Äù

Isinya: `/etc/passwd`, `/etc/fstab`, `/etc/hosts`, config Apache, SSH, dll.

Tempat favorit sysadmin buat ngoprek setting.

sudo nano /etc/fstab ‚Üí atur mount otomatis
/home
‚ÄúRumah pribadi tiap user.‚Äù

`/home/frijal/` ‚Üí folder kamu. Isi: dokumen, foto, `.bashrc`, `.config/`.

Root nggak punya rumah di sini. Dia tinggal di `/root`.

cd ~ ‚Üí langsung ke /home/nama-kamu
/lib
‚ÄúPerpustakaan buat program-program.‚Äù

File `.so` ‚Äî kayak DLL di Windows.

Tanpa `/lib`, banyak aplikasi error: *‚Äúlibrary not found‚Äù*.

ldd /bin/ls ‚Üí lihat library yang dipakai
/media
‚ÄúTempat flashdisk, CD, atau HDD eksternal numpang.‚Äù

Colok USB ‚Üí otomatis muncul di `/media/nama-kamu/USB-NAME`.

Bisa di-unmount aman lewat file manager.

/mnt
‚ÄúTempat numpang sementara, biasanya buat sysadmin.‚Äù

Dulu dipakai buat mount manual. Sekarang lebih sering `/media`.

Contoh: `mount /dev/sdb1 /mnt/backup`

/opt
‚ÄúRumah aplikasi dari luar (bukan dari repo resmi).‚Äù

Contoh: Google Chrome, VS Code, Zoom.

Struktur: `/opt/nama-app/`

/proc
‚ÄúCermin dari apa yang lagi jalan di sistem.‚Äù

**Bukan file beneran**, tapi virtual!

Contoh: `/proc/cpuinfo`, `/proc/meminfo`, `/proc/1234/` ‚Üí proses PID 1234.

cat /proc/uptime ‚Üí berapa lama sistem nyala
/root
‚ÄúRumahnya si Bos Besar: root.‚Äù

Bukan di `/home/root`, tapi langsung `/root`.

Jangan masuk kalau nggak perlu. Bahaya! ‚öÝÔ∏è

/run
‚ÄúData sementara yang cuma hidup pas sistem nyala.‚Äù

PID file, socket, status service. Hilang pas reboot.

Contoh: `/run/user/1000/` ‚Üí data user kamu saat login.

/sbin
‚ÄúPerintah khusus admin (superuser).‚Äù

Isi: `fdisk`, `reboot`, `ifconfig`, `fsck`.

Harus pake `sudo` atau login root.

/srv
‚ÄúData buat layanan (server).‚Äù

Contoh: `/srv/www/` ‚Üí file website, `/srv/ftp/` ‚Üí file FTP.

Jarang dipakai di desktop, tapi wajib di server.

/sys
‚ÄúJendela langsung ke kernel Linux.‚Äù

Mirip `/proc`, tapi lebih modern.

Bisa ubah setting kernel real-time.

echo 1 > /sys/class/leds/... ‚Üí nyalain lampu LED
/tmp
‚ÄúTempat sampah sementara.‚Äù

File di sini otomatis dihapus pas reboot.

Kadang penuh ‚Üí bikin sistem lemot.

/usr
‚ÄúPusat aplikasi dan data user.‚Äù

Strukturnya: `/usr/bin`, `/usr/lib`, `/usr/share`, `/usr/local`.

Besar banget, isinya ratusan MB sampai GB.

/var
‚ÄúData yang selalu berubah.‚Äù

Contoh: `/var/log/` ‚Üí log sistem, `/var/www/` ‚Üí web files.

Kalau penuh ‚Üí sistem bisa error!

sudo tail -f /var/log/syslog

Tips Praktis Buat Pemula Linux

  • **Jangan takut buka terminal:** `ls -la /` ‚Üí lihat semua folder root
  • **Coba jelajahi:** `cd /proc && cat cpuinfo` ‚Üí info CPU
  • **Backup dulu** sebelum edit `/etc` atau `/boot`
  • **Pake `tree`:** `sudo apt install tree && tree -L 2 /` ‚Üí lihat struktur rapi
  • **Baca man page:** `man hier` ‚Üí penjelasan resmi FHS (Filesystem Hierarchy Standard)
Linux itu kayak rumah besar:
`/home` = kamar pribadi
`/etc` = ruang kontrol
`/var` = gudang yang selalu bertambah
`/usr` = perpustakaan besar
`/tmp` = meja kerja sementara
`/root` = kamar bos (jangan masuk sembarangan!)

Semua punya fungsi, nggak ada yang sia-sia. Makanya Linux stabil ‚Äî karena **semua terstruktur!**

Kalau kamu baru belajar Linux, mulai dari:

ls -la /
