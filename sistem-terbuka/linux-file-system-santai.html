Sumber: Infografis Linux File System (2025)

Linux itu rapi banget ngatur filenya. Bayangin kayak rumah besar — semua punya tempatnya masing-masing!
/bin
“Tempatnya perintah-perintah dasar yang wajib ada.”

Isinya: `ls`, `cat`, `rm`, `cp`, `mv`, `echo`, dll.

Semua orang bisa pakai, bahkan saat sistem lagi “rusak parah”.

Kalau ini ilang? Ya... Linux bakal nangis. 😭

Contoh: /bin/ls, /bin/bash, /bin/mkdir
/boot
“Rumahnya file-file buat nyalain Linux.”

Ada: `vmlinuz` (kernel), `initramfs`, `grub.cfg`.

GRUB baca dari sini buat nampilin menu boot.

Jangan sembarangan edit, bisa-bisa laptop nggak nyala!

ls /boot → vmlinuz-6.5.0-14-generic
/dev
“Semua hardware di sini jadi 'file'.”

Contoh: `/dev/sda` = harddisk pertama, `/dev/null` = lubang hitam, `/dev/random` = angka acak.

Keren, kan? Linux bilang: **“Semua adalah file!”**

ls /dev → sda, null, random, tty, usb
/etc
“Pusat pengaturan sistem.”

Isinya: `/etc/passwd`, `/etc/fstab`, `/etc/hosts`, config Apache, SSH, dll.

Tempat favorit sysadmin buat ngoprek setting.

sudo nano /etc/fstab → atur mount otomatis
/home
“Rumah pribadi tiap user.”

`/home/frijal/` → folder kamu. Isi: dokumen, foto, `.bashrc`, `.config/`.

Root nggak punya rumah di sini. Dia tinggal di `/root`.

cd ~ → langsung ke /home/nama-kamu
/lib
“Perpustakaan buat program-program.”

File `.so` — kayak DLL di Windows.

Tanpa `/lib`, banyak aplikasi error: *“library not found”*.

ldd /bin/ls → lihat library yang dipakai
/media
“Tempat flashdisk, CD, atau HDD eksternal numpang.”

Colok USB → otomatis muncul di `/media/nama-kamu/USB-NAME`.

Bisa di-unmount aman lewat file manager.

/mnt
“Tempat numpang sementara, biasanya buat sysadmin.”

Dulu dipakai buat mount manual. Sekarang lebih sering `/media`.

Contoh: `mount /dev/sdb1 /mnt/backup`

/opt
“Rumah aplikasi dari luar (bukan dari repo resmi).”

Contoh: Google Chrome, VS Code, Zoom.

Struktur: `/opt/nama-app/`

/proc
“Cermin dari apa yang lagi jalan di sistem.”

**Bukan file beneran**, tapi virtual!

Contoh: `/proc/cpuinfo`, `/proc/meminfo`, `/proc/1234/` → proses PID 1234.

cat /proc/uptime → berapa lama sistem nyala
/root
“Rumahnya si Bos Besar: root.”

Bukan di `/home/root`, tapi langsung `/root`.

Jangan masuk kalau nggak perlu. Bahaya! ⚠️

/run
“Data sementara yang cuma hidup pas sistem nyala.”

PID file, socket, status service. Hilang pas reboot.

Contoh: `/run/user/1000/` → data user kamu saat login.

/sbin
“Perintah khusus admin (superuser).”

Isi: `fdisk`, `reboot`, `ifconfig`, `fsck`.

Harus pake `sudo` atau login root.

/srv
“Data buat layanan (server).”

Contoh: `/srv/www/` → file website, `/srv/ftp/` → file FTP.

Jarang dipakai di desktop, tapi wajib di server.

/sys
“Jendela langsung ke kernel Linux.”

Mirip `/proc`, tapi lebih modern.

Bisa ubah setting kernel real-time.

echo 1 > /sys/class/leds/... → nyalain lampu LED
/tmp
“Tempat sampah sementara.”

File di sini otomatis dihapus pas reboot.

Kadang penuh → bikin sistem lemot.

/usr
“Pusat aplikasi dan data user.”

Strukturnya: `/usr/bin`, `/usr/lib`, `/usr/share`, `/usr/local`.

Besar banget, isinya ratusan MB sampai GB.

/var
“Data yang selalu berubah.”

Contoh: `/var/log/` → log sistem, `/var/www/` → web files.

Kalau penuh → sistem bisa error!

sudo tail -f /var/log/syslog

Tips Praktis Buat Pemula Linux

  • **Jangan takut buka terminal:** `ls -la /` → lihat semua folder root
  • **Coba jelajahi:** `cd /proc && cat cpuinfo` → info CPU
  • **Backup dulu** sebelum edit `/etc` atau `/boot`
  • **Pake `tree`:** `sudo apt install tree && tree -L 2 /` → lihat struktur rapi
  • **Baca man page:** `man hier` → penjelasan resmi FHS (Filesystem Hierarchy Standard)
Linux itu kayak rumah besar:
`/home` = kamar pribadi
`/etc` = ruang kontrol
`/var` = gudang yang selalu bertambah
`/usr` = perpustakaan besar
`/tmp` = meja kerja sementara
`/root` = kamar bos (jangan masuk sembarangan!)

Semua punya fungsi, nggak ada yang sia-sia. Makanya Linux stabil — karena **semua terstruktur!**

Kalau kamu baru belajar Linux, mulai dari:

ls -la /
