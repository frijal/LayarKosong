Mengupas Hasil 'apt --audit': Arti & Solusi Lengkapnya

"Penjelasan kamu bagus." Berawal dari kalimat itu, mari kita ubah hasil audit teknis di terminal Linux menjadi artikel yang mudah dipahami semua orang!

Dipublikasikan pada 12 Oktober 2025

Hasil Auditnya Keluar! Apa Kata Terminal?

Setelah menjalankan perintah sudo apt update --audit, terminal kita jadi lebih "cerewet" dari biasanya. Dia nggak cuma bilang "semua paket sudah update", tapi juga memberikan laporan audit lengkap yang informatif banget. Kira-kira, beginilah penampakannya:

Warning: http://download.opensuse.org/.../InRelease: Policy will reject signature within a year...
Audit: http://.../InRelease: Sub-process /usr/bin/sqv returned an error code (1), error message is:
   Error: Policy rejected packet type
   Caused by:
       Signature Packet v3 is not considered secure since 2026-02-01T00:00:00Z
Audit: The sources.list(5) entry for 'http://ftp.es.debian.org/debian' should be upgraded to deb822 .sources
Audit: Missing Signed-By in the sources.list(5) entry for 'http://ftp.es.debian.org/debian'
... (pesan serupa berulang kali) ...
Audit: Some sources can be modernized. Run 'apt modernize-sources' to do so.

Kelihatannya rumit ya? Sebenarnya nggak juga. Laporan ini cuma ngasih tau kita dua masalah utama: satu dari luar (penyedia software) dan satu dari dalam (konfigurasi kita sendiri).

Bedah Hasil Audit (Satu per Satu)

Mari kita kupas laporan ini jadi dua bagian utama biar gampang dicerna.

Masalah #1: Tanda Tangan Digital Versi Jadul

Pesan Signature Packet v3 is not considered secure since 2026-02-01 adalah biang keladi dari peringatan awal. Artinya, repositori dari OpenSuse itu masih pakai "segel keamanan" versi lama (v3). Standar keamanan modern akan menganggap segel ini tidak aman lagi mulai 1 Februari 2026. Ini murni masalah di sisi mereka, dan kita hanya bisa berharap mereka segera memperbaruinya.

Masalah #2: Konfigurasi Kita yang Perlu Dimodernisasi

Pesan yang paling banyak muncul adalah should be upgraded to deb822 .sources dan Missing Signed-By. Ini artinya cara kita mendaftarkan sumber-sumber software di sistem (file sources.list) masih pakai format lawas. Format baru yang disarankan, yaitu deb822, jauh lebih aman dan rapi karena setiap repositori ditautkan langsung ke kunci keamanannya masing-masing. Ini penting untuk mencegah celah keamanan.

Solusi Pamungkas: Modernisasi! âœ¨

Nah, bagian paling keren dari audit ini adalah pesan terakhirnya: Run 'apt modernize-sources' to do so. Betul sekali, sistem apt sudah menyiapkan perintah otomatis untuk membereskan semua ini!

Ikuti langkah-langkah mudah dan aman berikut:

Langkah 1: Backup Dulu Biar Tenang (Opsional)

Untuk jaga-jaga, kita backup dulu konfigurasi yang lama. Buka terminal dan ketik:

sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
sudo cp -r /etc/apt/sources.list.d/ /etc/apt/sources.list.d.bak

Langkah 2: Eksekusi Perintah Modernisasi!

Ini dia langkah utamanya. Jalankan perintah sakti yang disarankan oleh sistem:

sudo apt modernize-sources

Perintah ini akan secara otomatis mengubah semua file konfigurasi .list lama Anda menjadi format .sources (deb822) yang baru dan lebih aman.

Langkah 3: Verifikasi dan Nikmati Hasilnya

Setelah selesai, jalankan update sekali lagi untuk melihat perubahannya:

sudo apt update

Voila! Seharusnya semua peringatan audit tentang format deb822 dan Missing Signed-By kini telah lenyap. Sistem Anda jadi lebih bersih, modern, dan aman. Peringatan soal OpenSuse mungkin masih ada, tapi setidaknya bagian kita sudah beres!
