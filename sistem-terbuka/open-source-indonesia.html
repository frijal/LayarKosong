Open Source vs Gratis: Paham Lisensi, Maksimalkan Peluang

Di Indonesia, banyak yang membutuhkan aplikasi "gratis," tetapi belum tentu paham bedanya dengan "open source." Mari kita urai perbedaannya, kenali jenis lisensi, dan lihat bagaimana open source bisa menjadi peluang usaha yang etis.

Gratis Belum Tentu Open Source

Aplikasi gratis (freeware) dapat digunakan tanpa membayar, tetapi kode sumbernya tertutup. Anda tidak bisa melihat, memodifikasi, atau mendistribusikan ulang kodenya. Sebaliknya, open source mewajibkan penyediaan kode sumber dan memberikan kebebasan untuk memodifikasi serta mendistribusikannya kembali, sesuai dengan syarat lisensinya.

Jenis Lisensi Populer dan Implikasinya

  • GPL (Copyleft Kuat): Membolehkan penggunaan komersial, tetapi setiap produk turunan yang didistribusikan wajib tetap berlisensi GPL dan membuka kode sumbernya.
  • LGPL: Mirip GPL, tetapi lebih longgar untuk library. Anda bisa menautkan library LGPL ke aplikasi komersial tanpa harus membuka seluruh kode aplikasi Anda.
  • Apache 2.0: Sangat ramah komersial. Anda hanya perlu menyediakan atribusi dan pemberitahuan lisensi.
  • MIT/BSD: Lisensi yang sangat permisif. Cukup sertakan pemberitahuan hak cipta dan lisensi dalam produk Anda.

Model Usaha Berbasis Open Source

Memanfaatkan open source untuk bisnis adalah hal yang legal dan etis. Beberapa model yang populer di antaranya:

  • Layanan (Services): Menawarkan jasa implementasi, kustomisasi, dan migrasi sistem berbasis open source.
  • Dukungan (Support): Menyediakan paket dukungan teknis (SLA), pemeliharaan, dan pelatihan.
  • Hosted/SaaS: Menyediakan layanan terkelola (cloud) dari perangkat lunak open source.
Praktik Baik: Selalu simpan daftar dependensi dan lisensinya, sertakan file `NOTICE` jika diperlukan, dokumentasikan perubahan, dan berikan atribusi yang benar kepada pembuat aslinya.

Contoh File NOTICE

Jika proyek komersial Anda menggunakan komponen open source, Anda mungkin perlu menyertakan file NOTICE seperti ini:

NOTICE
This product includes software developed by various open source communities.
Components:
- Library A (Apache-2.0) 🄯 2018–2025 Example Authors
- Library B (MIT) 🄯 2016–2025 Another Authors
Changes:
- Modified module X for performance.
