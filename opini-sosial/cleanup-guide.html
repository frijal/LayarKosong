🪄 Petunjuk Penggunaan Cleanup Script

Skrip `ext/cleanup.js` ini berfungsi untuk **membersihkan dependensi Node.js yang sudah tidak terpakai** dalam proyek GitHub Pages kamu. Dengan menggunakan paket `depcheck` dan `inquirer`, kamu bisa memeriksa paket yang tidak lagi dibutuhkan dan menghapusnya dengan aman.

📦 Persiapan Awal

Langkah pertama, pastikan kamu sudah memasang dua paket pendukung sekali saja:

npm install --save-dev depcheck inquirer

Kedua paket ini membantu skrip melakukan pemeriksaan otomatis serta menampilkan antarmuka pilihan di terminal agar kamu bisa memilih paket yang ingin dihapus.

🚀 Cara Menjalankan Skrip

Simpan file berikut di dalam folder `ext/` proyek kamu dengan nama `cleanup.js`.

Lalu jalankan dari terminal:

node ext/cleanup.js

Setelah dijalankan, daftar paket yang tidak terpakai akan muncul. Tekan **spasi** untuk memilih paket yang ingin dihapus, lalu tekan **Enter**.

🧹 Reset Total (Opsional)

Setelah memilih paket, kamu akan ditanya apakah ingin menghapus folder `node_modules` dan menginstal ulang semua dependensi. Pilih **Ya** untuk melakukan *full reset*—berguna ketika ingin memastikan lingkungan proyek benar-benar bersih.

Hapus node_modules & reinstall? (Y/n)

Jika kamu memilih **Ya**, maka:

  • Folder `node_modules` akan dihapus.
  • File `package-lock.json` juga ikut dihapus.
  • Semua dependensi akan diinstal ulang otomatis menggunakan `npm install`.

⚡ Shortcut di package.json

Biar lebih cepat, kamu bisa menambahkan shortcut ini di dalam file `package.json`:

"scripts": {
  "clean": "node ext/cleanup.js"
}

Sehingga cukup jalankan perintah ini dari terminal:

npm run clean

Sangat membantu ketika kamu sering melakukan pembersihan otomatis sebelum build ulang proyek.
