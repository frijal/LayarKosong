Kenapa Kita Butuh Tunnel? (Why)

Masalah utama server rumahan adalah: **ISP kita pelit!** Kebanyakan ISP memberikan IP Private (CGNAT) yang bikin server kita nggak bisa "dilihat" dari luar. Dulu solusinya pakai Port Forwarding, tapi itu bahaya banget karena sama aja buka pintu rumah lebar-lebar buat maling digital.

**Cloudflare Tunnel** bekerja kayak jalan pintas rahasia. Server kamu yang ada di Debian bakal bikin koneksi aman (outbound) ke Cloudflare, dan Cloudflare bakal nyambungin itu ke domain kamu. Hasilnya? Aman, nggak perlu buka port router, dan bisa nembus blokir ISP!

Apa Saja Resepnya? (What)

  • **Nextcloud** yang sudah jalan di Docker (dari tutorial sebelumnya).
  • **Domain:** Bisa pakai domain berbayar (.com/.id) atau cari yang gratisan (seperti dari Freenom jika masih tersedia, atau subdomain dari penyedia lain).
  • Akun gratis di **Cloudflare**.

Langkah-Langkah Eksekusi (How)

1. Setup di Dashboard Cloudflare

Login ke dashboard Cloudflare kamu, masuk ke menu **Zero Trust** > **Networks** > **Tunnels**. Klik "Create a Tunnel", kasih nama (misal: "Server-Debian-Balikpapan"), lalu simpan.

Kamu bakal dapet sebuah **Token** panjang. Simpan token itu, jangan kasih tahu siapa-siapa!

2. Update Docker Compose

Kita bakal tambahin satu layanan lagi di file `docker-compose.yml` yang kemarin kita buat. Jadi Cloudflare bakal jalan berdampingan sama Nextcloud kamu.

nano ~/nextcloud-server/docker-compose.yml

Tambahkan service `tunnel` di bagian bawah:

  tunnel:
    container_name: cloudflared-tunnel
    image: cloudflare/cloudflared:latest
    restart: always
    command: tunnel run --token MASUKKAN_TOKEN_KAMU_DISINI

3. Restart Docker Compose

Update kontainer kamu biar layanan baru ini jalan:

docker compose up -d

4. Hubungkan Domain

Kembali ke dashboard Cloudflare, di tab **Public Hostname**, klik "Add a public hostname":

  • **Subdomain:** cloud (atau terserah kamu)
  • **Domain:** pilih domain kamu
  • **Service Type:** HTTP
  • **URL:** app:80 (karena nama service Nextcloud kita kemarin adalah 'app')

5. Setting "Trusted Domains" di Nextcloud

Ini langkah krusial! Nextcloud secara default bakal nolak akses dari domain baru demi keamanan. Kamu harus kasih tahu kalau domain kamu itu aman.

sudo docker exec --user www-data nextcloud-server-app-1 php occ config:system:set trusted_domains 1 --value=cloud.domainkamu.com

**Ganti `nextcloud-server-app-1` dengan nama kontainer app kamu dan masukkan domain aslimu.*

Kapan Ini Bisa Digunakan? (When)

Detik ini juga! Begitu semua selesai, kamu bisa coba matikan Wi-Fi di HP kamu, pakai paket data, lalu buka `cloud.domainkamu.com` di browser. Kalau muncul halaman login Nextcloud, selamat! Kamu sudah punya Cloud pribadi yang bisa diakses dari tengah hutan Kalimantan sekalipun, asal ada sinyal.
