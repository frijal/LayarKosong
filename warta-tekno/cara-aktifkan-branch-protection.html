Cara Mengaktifkan Branch Protection di GitHub

Panduan ini akan menunjukkan cara mengaktifkan **branch protection** di GitHub. Fitur ini sangat penting untuk memastikan semua perubahan pada branch utama (seperti `main` atau `master`) melalui proses review via **Pull Request (PR)**.

1. Masuk ke Pengaturan Repositori

Buka repositori Anda di GitHub, lalu klik tab **Settings**.

2. Pilih Menu Branches

Di menu navigasi sebelah kiri, pilih **Branches**. Kemudian, klik tombol **Add branch protection rule**.

3. Tentukan Branch yang Akan Dilindungi

Pada kolom `Branch name pattern`, ketik nama branch yang ingin Anda lindungi, misalnya `main`. Anda juga bisa menggunakan wildcard seperti `release/*` untuk melindungi semua branch yang diawali dengan "release/".

4. Aktifkan Aturan Proteksi

Centang opsi-opsi yang Anda butuhkan. Beberapa yang paling umum adalah:

  • **Require a pull request before merging:** Ini adalah aturan inti. Semua perubahan wajib melalui PR.
  • **Require status checks to pass before merging:** Berguna jika Anda memiliki workflow CI/CD. PR tidak bisa di-merge jika ada tes yang gagal.
  • **Include administrators:** Menerapkan aturan ini bahkan untuk admin repositori.

5. Simpan Aturan

Klik tombol **Create** atau **Save changes** di bagian bawah halaman. Setelah ini, setiap upaya untuk melakukan `push` langsung ke branch `main` akan diblokir, dan semua perubahan harus diajukan melalui Pull Request.
