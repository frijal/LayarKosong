Branches di GitHub: Penjelasan Santai, Lengkap, dan Praktis

Branch (cabang) adalah jalur pengembangan terpisah di dalam sebuah repositori Git. Ibaratnya, Anda punya pohon kode: batang utamanya adalah branch `main`, dan Anda bisa membuat cabang baru untuk mengerjakan fitur atau memperbaiki bug tanpa mengganggu batang utama.

Fungsi dan Manfaat Branch

  • Isolasi Pekerjaan: Anda bisa mengerjakan fitur baru tanpa merusak kode utama yang stabil.
  • Kolaborasi Aman: Setiap developer bisa bekerja di branch masing-masing, lalu menggabungkannya melalui Pull Request.
  • Eksperimen Bebas Risiko: Jika sebuah ide gagal, Anda cukup menghapus branch tersebut tanpa ada dampak ke kode utama.

Alur Kerja Umum

  1. Buat branch baru dari `main`:
    git checkout -b fitur-login
  2. Kerjakan perubahan di branch tersebut, lalu commit pekerjaan Anda.
  3. Push branch baru ke GitHub:
    git push -u origin fitur-login
  4. Buat Pull Request di GitHub untuk menggabungkan perubahan Anda ke `main`.
