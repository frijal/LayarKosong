Panduan Migrasi Node.js ke Bun

5 Menit Baca | Fakhrul Rijal

Kenalan Dulu: Prinsip 5W+1H Migrasi Bun

Tips E-E-A-T

Sebagai praktisi yang sering berkutat dengan optimasi sistem, aku menyarankan migrasi ini terutama jika proyek kamu memiliki banyak *dependencies*. Kecepatan Bun dalam menangani ribuan file akan sangat menghemat waktu berharga kamu.

Langkah 1: Instalasi Bun

Buka terminal andalan kamu, lalu tempel perintah sakti ini:

curl -fsSL https://bun.sh/install | bash

Setelah itu, restart terminal atau jalankan `source ~/.bashrc` (atau zshrc) agar perintah `bun` bisa dikenali.

Langkah 2: Mengubah package.json

Sebenarnya, kamu tidak perlu mengubah isi file secara drastis. Cukup pastikan bagian `scripts` menggunakan perintah Bun agar performanya maksimal.

{
 "name": "proyek-keren-kamu",
 "type": "module",
 "scripts": {
 "start": "bun index.ts",
 "dev": "bun --watch index.ts",
 "test": "bun test"
 }
}

Gunakan flag `--watch` untuk fitur *hot reload* otomatis, jadi kamu tidak perlu restart manual tiap kali simpan file. Sat-set banget, kan?

Langkah 3: Ganti Lockfile

Lockfile (seperti `package-lock.json` atau `yarn.lock`) adalah catatan versi paket yang kamu pakai. Di dunia Bun, kita pakai yang lebih ringan.

  1. Hapus file lama: `rm package-lock.json`.
  2. Jalankan perintah: `bun install`.
  3. Boom! Kamu sekarang punya `bun.lock` yang ukurannya jauh lebih kecil dan efisien.

Optimasi Core Web Vitals

Migrasi ke Bun secara tidak langsung membantu optimasi **LCP (Largest Contentful Paint)** pada alur kerja build statis. Dengan waktu build yang lebih cepat, proses *deployment* konten baru jadi lebih singkat. Selain itu, Bun sangat hemat memori, yang berarti server kamu punya lebih banyak ruang napas untuk melayani pengunjung dengan cepat.
