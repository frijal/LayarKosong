Perubahan Epochal! Git 3.0 Ganti 'master' Jadi 'main' sebagai Default Branch! 🥳

Ada kabar **super penting** nih dari dunia Git yang wajib banget kamu tahu. Versi terbaru Git, yaitu **Git 3.0**, bakal membawa perubahan fundamental yang sudah lama dinanti-nanti. Intinya: Kita bakal **selamat tinggal pada branch default 'master'** dan menyambut **'main'** sebagai gantinya! 🎊

Mulai Git 3.0 ke depan, kamu nggak perlu repot-repot lagi *ngeset* branch default buat repository baru. Secara otomatis, sistem Git akan langsung menggunakan **'main'**! Ini adalah langkah kecil yang punya makna besar, lho. Perubahan ini juga sejalan dengan upaya komunitas *developer* global untuk menggunakan terminologi yang lebih inklusif dan netral. Keren kan? 😎

Starting with Git 3.0, developers will no longer have to configure the default branch for new repositories.

Pengumuman resminya sudah nongol di Git 2.52 yang dirilis minggu ini. Perhatikan baik-baik *patch notes*-nya, ada baris kecil tapi bermakna yang bilang begini:

Declare that “git init” that is not otherwise configured uses ‘main’ as the initial branch, not ‘master’, starting Git 3.0.

Secara bahasa santai: **"Mulai Git 3.0, perintah 'git init' yang belum dikonfigurasi akan pakai 'main' sebagai branch awal, bukan 'master'."** Sudah jelas banget kan? Ini menunjukkan bahwa perubahan ini sudah dikunci dan tinggal menunggu tanggal rilis resminya.

Perubahan ini memang bukan barang baru. Proyek Git, yang berada di bawah naungan Software Freedom Conservancy, sudah mengumumkan niat mereka untuk mengganti nama branch default sejak 23 Juni 2020. Platform *hosting* repo raksasa seperti GitHub juga bergerak cepat, mereka sudah **mengubah branch default menjadi main** untuk *repository* baru sejak 1 Oktober 2020. Jadi, bisa dibilang ini adalah langkah konsolidasi dari Git untuk mengikuti tren yang sudah diadopsi oleh ekosistemnya. 👍

Sayangnya, Git 3.0 belum punya tanggal rilis pasti saat ini. Tapi, kabar burung (atau lebih tepatnya, *current estimates*) memprediksi bahwa kita mungkin baru bisa menikmati versi *major* ini menjelang **akhir tahun 2026**. Masih lama ya? Tapi setidaknya kita tahu apa yang harus dinantikan! **Bersabar *is a virtue*, teman-teman!** 🙏

Bukan Cuma Nama! Fitur Keren Lain di Git 3.0

Perubahan ke *branch* main hanyalah salah satu *highlight* saja. Git 3.0 juga membawa perubahan *breaking changes* penting lainnya (cek [di sini](https://git-scm.com/docs/BreakingChanges.html) untuk detailnya) yang bakal bikin kerjaan kita makin aman dan *ngebut*. Beberapa di antaranya adalah:

  • **Keamanan Lebih Kuat:** Fungsi *hash* default akan diganti dari **SHA-1** ke **SHA-256**. Ini adalah *upgrade* keamanan yang signifikan. SHA-1 sudah dianggap rentan, jadi pindah ke SHA-256 adalah langkah *pro* banget! 🛡️
  • **Performa dan Kompatibilitas:** Ada perubahan format penyimpanan default untuk *support* yang lebih baik di sistem operasi seperti **macOS dan Windows**, sekaligus meningkatkan performa secara keseluruhan. Siapa sih yang gak mau Git-nya makin cepat? 🚀
  • **Integrasi Rust:** **Rust** akan diintegrasikan secara lebih formal ke dalam proses *build* Git. Ini bisa berarti peningkatan keamanan dan performa karena Rust terkenal dengan *memory safety*-nya.

Singkatnya, Git 3.0 bukan sekadar *ganti nama*. Ini adalah pembaruan besar yang berfokus pada **keamanan, kinerja, dan inklusivitas**. Mari kita nantikan bersama kedatangannya sambil terus *ngoding* dengan *branch* main di proyek-proyek baru kita! Selamat *ngoding*! 💻
