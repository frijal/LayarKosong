Jenis Kode Error Website Umum

Kode status HTTP adalah sinyal dari server tentang hasil permintaan browser. Secara umum, 4xx berkaitan dengan request, akses, atau alamat dari sisi klien, sedangkan 5xx menandakan gangguan di sisi server.

400 Bad Request

Request-nya nggak kebaca server. Bisa jadi URL typo, cookie/cache lagi rewel, atau data yang dikirim udah kacau.

💡 Cek lagi alamat URL-nya. Kalau masih ngambek, coba bersihin cookie browser.

401 Unauthorized

Halaman ini butuh login dulu. Bisa jadi kamu belum masuk, sesi login udah basi, atau username/password-nya salah.

💡 Login ulang pelan-pelan, sabar, jangan kayak orang lagi ngejar diskon flash sale.

403 Forbidden

Server paham kamu mau ke mana, tapi dia bilang: “Maaf, kamu nggak boleh masuk.”

💡 Biasanya karena aksesmu nggak cukup, permission ngaco, atau alamat IP kamu kena blokir.

404 Not Found

Halamannya nggak ketemu. Salah ketik URL, dihapus, atau pindah rumah tanpa pamit.

💡 Cek lagi URL-nya. Siapa tahu cuma salah satu huruf, tapi dramanya satu halaman.

408 Request Timeout

Server kelamaan nunggu browser ngirim request. Ibarat janjian, tapi yang satu nggak jadi muncul.

💡 Coba reload pas koneksi udah stabil. Kalau WiFi-nya lagi melamun, tunggu dulu bentar.

429 Too Many Requests

Kebanyakan request dalam waktu mepet. Server mulai curiga kamu bot, scraper, atau manusia yang terlalu semangat.

💡 Santai bentar. Tunggu beberapa menit, baru coba lagi.

500 Internal Server Error

Servernya error dari dalam. Bisa script ngadat, plugin ribut, database ngambek, atau konfigurasi lagi berantakan.

💡 Ini wilayah admin/server. Pengunjung cuma bisa menatap layar sambil berharap.

502 Bad Gateway

Server perantara menerima jawaban aneh dari server lain. Jadi bukan salah kamu, rantai belakangnya yang lagi kusut.

💡 Biasanya proxy, CDN, atau backend lagi gagal ngobrol dengan benar.

503 Service Unavailable

Server lagi nggak sanggup melayani. Bisa karena maintenance, sibuk, atau resource udah megap-megap.

💡 Servernya tepar. Balik lagi nanti sore, jangan dipaksa kayak laptop tua buka browser 38 tab.

504 Gateway Timeout

Gateway kelamaan nunggu server utama. Server belakangnya telat jawab sampai proxy keburu nyerah.

💡 Biasanya karena backend lemot, database berat, atau koneksi antarserver lagi nggak akur.
