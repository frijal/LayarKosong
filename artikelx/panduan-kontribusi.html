‚öôÔ∏è Bagaimana Cara Berkontribusi?

Nah, ini dia bagian yang paling seru! Ada banyak banget cara untuk membantu proyek Layar Kosong, dan yang menariknya, nggak harus selalu lewat kode! Yup, kamu bisa berkontribusi dalam berbagai bentuk. Yuk kita bahas satu per satu!

üêõ 1. Pelaporan Bug (Masalah)

Menemukan sesuatu yang error atau nggak beres? Workflow otomatisnya bermasalah? Template HTML-nya rusak? Atau mungkin ada tautan yang broken? Wah, ini kesempatan bagus untuk berkontribusi!

Melaporkan bug itu sebenarnya bentuk kontribusi yang sangat berharga, lho. Karena dengan laporan bug yang detail, developer lain bisa lebih mudah menemukan dan memperbaiki masalahnya. Jadi, jangan sungkan untuk melaporkan kalau nemuin sesuatu yang aneh!

Cara Melaporkan Bug yang Baik

  • Gunakan tab Issues di GitHub untuk membuat laporan baru
  • Kasih deskripsi yang jelas ‚Äî apa yang terjadi, kapan terjadi, dan di mana terjadinya
  • Kalau bisa, sertakan langkah-langkah untuk mereproduksi bug tersebut (ini super helpful!)
  • Tambahkan screenshot atau screen recording kalau memungkinkan
  • Gunakan label bug biar mudah dikategorikan

üíª 3. Kontribusi Kode (Pull Request)

Nah, ini dia untuk kamu yang pengin terjun langsung ke kode! Entah itu perbaikan bug, pembaruan workflow YAML, peningkatan style CSS, atau penambahan fitur baru ‚Äî semuanya disambut dengan hangat!

. Persiapan Sebelum Mulai

Sebelum mulai coding, ada beberapa langkah persiapan yang perlu kamu lakukan. Ini penting biar kontribusimu bisa di-merge dengan lancar tanpa kendala teknis.

  1. Fork Repository ‚Äî Pertama, fork dulu repository Layar Kosong ke akun GitHub-mu. Ini akan membuat salinan repository di akun kamu sendiri.
  2. Clone ke Lokal ‚Äî Setelah di-fork, clone repo tersebut ke komputer lokalmu menggunakan git clone.
  3. Buat Branch Baru ‚Äî Jangan langsung kerja di branch main! Buat branch baru dengan nama yang deskriptif, misalnya feat/perbaikan-css atau fix/workflow-bug.

. Aturan Penulisan Kode

Biar kode yang kamu tulis konsisten dengan yang sudah ada dan mudah di-review, perhatikan beberapa aturan berikut:

HTML/Markdown

Prioritaskan kesederhanaan dan keterbacaan. Pastikan kode HTML-mu valid, bersih, dan well-structured. Hindari nesting yang terlalu dalam dan gunakan semantic elements yang tepat.

CSS

Karena ini proyek static site yang super ringan, usahakan pakai CSS yang efisien. Kalau cuma perlu style sederhana, inline atau scoped CSS bisa jadi pilihan yang oke. Hindari CSS yang bloated!

Workflow (.yml)

Untuk workflow YAML, pastikan skrip yang kamu buat atau ubah itu berfungsi dengan benar dan aman. Test dulu di fork-mu sebelum submit PR. Keamanan itu prioritas utama!

. Mengirimkan Pull Request (PR)

Udah selesai coding? Mantap! Sekarang saatnya kirimkan Pull Request. Tapi sebelum itu, pastikan beberapa hal berikut:

  • Commit message yang jelas ‚Äî Gunakan format yang deskriptif, misalnya: fix: Perbaiki link favicon yang rusak atau feat: Tambahkan action ping sitemap. Ini membantu reviewer memahami konteks perubahanmu.
  • Buka PR ke branch main ‚Äî Target PR-mu adalah branch main dari repository utama (bukan fork-mu).
  • Deskripsikan dengan detail ‚Äî Di deskripsi PR, jelaskan: Apa yang kamu ubah? Kenapa kamu mengubahnya? Apa hasil yang diharapkan? Semakin detail, semakin cepat proses review-nya!
‚úÖ

Pro tip: Sebelum submit PR, pastikan kamu sudah pull perubahan terbaru dari upstream repository. Ini menghindari konflik merge yang bikin pusing!

üìù Kontribusi Konten (Artikel)

Nggak jago coding tapi jago nulis? Tenang, kamu tetap bisa berkontribusi kok! Karena Layar Kosong fokus pada publikasi konten HTML, kami sangat menyambut baik kontribusi berupa artikel atau tulisan.

Mau nulis tutorial? Essay? Review teknologi? Atau apapun yang bermanfaat? Gas aja! Berikut langkah-langkahnya:

  1. Buat file HTML baru ‚Äî Namai file-mu dengan format yang SEO-friendly, misalnya artikel-judul-mu.html. Gunakan huruf kecil dan strip sebagai pemisah kata.
  2. Letakkan di folder artikelx/ ‚Äî Semua artikel kontribusi ditempatkan di folder khusus bernama artikelx/. Ini memudahkan pengelolaan dan review.
  3. Kirimkan Pull Request ‚Äî Sama seperti kontribusi kode, kirimkan PR dengan deskripsi yang jelas tentang artikel yang kamu tulis.

‚ö†Ô∏è Penting Banget Mengenai Lisensi Konten: Karena proyek ini menggunakan UNLICENSE (Public Domain), dengan mengirimkan artikel ke folder artikelx/, kamu secara otomatis melepaskan hak cipta atas tulisanmu tersebut ke Domain Publik. Ini artinya siapa pun bebas menggunakan, memodifikasi, dan mendistribusikan tulisanmu tanpa perlu izin. Pastikan kamu benar-benar nyaman dan setuju dengan hal ini sebelum submit ya!
