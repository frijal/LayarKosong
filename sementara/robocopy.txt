  <header>
    <h1>Panduan Lengkap Robocopy untuk Sync, Backup, dan Optimasi</h1>
    <p>Halaman ini menggabungkan seluruh percakapan: sintaks Robocopy, solusi error auditing, opsi yang direkomendasikan, file .bat, penjadwalan Task Scheduler, perbandingan dengan rsync, optimasi performa, dan pertimbangan transfer lewat WLAN.</p>
  </header>

  <section>
    <h2>Ringkasan Percakapan</h2>
    <p>Topik yang dibahas meliputi: penulisan sintaks Robocopy yang benar untuk UNC share, mengatasi error terkait auditing (/COPYALL), memilih opsi untuk hanya menyalin file tanpa menimpa file yang lebih baru, menjalankan beberapa perintah Robocopy dalam satu .bat, perbandingan Robocopy vs rsync, pilihan menjalankan .bat lewat CMD atau PowerShell, serta optimasi performa untuk banyak file dan file besar lewat WLAN.</p>
  </section>

  <section>
    <h2>Perintah Robocopy yang Direkomendasikan</h2>
    <h3>Perintah dasar sesuai kebutuhan Anda</h3>
    <pre><code>robocopy "\\192.168.9.166\D\Data MMP\7. Store File" "C:\Users\Server\Documents\rijal\7. Store File" /E /XO /W:0 /R:0 /COPY:DAT /DCOPY:DAT /MT:16</code></pre>

    <h3>Penjelasan opsi penting</h3>
    <ul>
      <li><strong>/E</strong> — salin semua subfolder termasuk yang kosong.</li>
      <li><strong>/XO</strong> — exclude older; jangan timpa file di destination yang lebih baru.</li>
      <li><strong>/W:0</strong> — waktu tunggu 0 detik sebelum retry.</li>
      <li><strong>/R:0</strong> — tidak ada retry; file yang gagal langsung dilewati.</li>
      <li><strong>/COPY:DAT</strong> — hanya salin Data, Attributes, Timestamp (tanpa ACL, owner, auditing).</li>
      <li><strong>/DCOPY:DAT</strong> — salin atribut folder dasar.</li>
      <li><strong>/MT:16</strong> — multi-threading 16 thread untuk percepatan (sesuaikan).</li>
    </ul>

    <div class="note">
      <strong>Catatan</strong>: Hindari <strong>/COPYALL</strong> jika Anda tidak memiliki hak Manage Auditing and Security Log; gunakan <strong>/COPY:DAT</strong> untuk menghindari error auditing.
    </div>
  </section>

  <section>
    <h2>Menangani Error Auditing</h2>
    <p>Jika muncul pesan:</p>
    <pre><code>ERROR : You do not have the Manage Auditing user right.
You need this to copy auditing information (/COPY:U or /COPYALL).</code></pre>

    <p>Solusi praktis:</p>
    <ul>
      <li>Jalankan Command Prompt sebagai Administrator jika memang perlu menyalin auditing.</li>
      <li>Jika tidak perlu auditing, ganti <code>/COPYALL</code> dengan <code>/COPY:DAT</code>.</li>
      <li>Jika tetap ingin menyalin auditing, tambahkan hak <strong>Manage Auditing and Security Log</strong> ke akun yang menjalankan task melalui Local Security Policy.</li>
    </ul>
  </section>

  <section>
    <h2>File Batch (.bat) dengan Beberapa Perintah Robocopy</h2>
    <p>Contoh file <code>backup.bat</code> yang menjalankan dua perintah Robocopy berurutan dan menulis log terpisah:</p>
    <pre><code>@echo off
rem Backup dari server ke folder lokal
robocopy "\\192.168.9.166\D\Data MMP\7. Store File" "C:\Users\Server\Documents\rijal\7. Store File" /E /XO /W:0 /R:0 /COPY:DAT /DCOPY:DAT /MT:16 /LOG:C:\Logs\robocopy_store.log

rem Backup folder Dyah File
robocopy "\\192.168.9.166\D\Data MMP\9. Dyah File" "C:\Users\Server\Documents\rijal\9. Dyah File" /E /XO /W:0 /R:0 /COPY:DAT /DCOPY:DAT /MT:16 /LOG:C:\Logs\robocopy_dyah.log

echo Semua backup selesai!
pause</code></pre>

    <ul>
      <li>Gunakan folder log terpisah (mis. <code>C:\Logs</code>).</li>
      <li>Hapus <code>pause</code> jika dijalankan otomatis lewat Task Scheduler.</li>
      <li>Pastikan akun yang menjalankan task memiliki akses ke UNC share.</li>
    </ul>
  </section>

  <section>
    <h2>Menambahkan Timestamp ke Log</h2>
    <p>Contoh menambahkan header waktu ke file log per-run:</p>
    <pre><code>@echo off
set LOGDIR=C:\Logs
set LOGFILE=%LOGDIR%\robocopy_all_%date:~-4,4%%date:~-10,2%%date:~-7,2%_%time:~0,2%%time:~3,2%.log
if not exist "%LOGDIR%" mkdir "%LOGDIR%"

echo ============================== >> "%LOGFILE%"
echo Run at %date% %time% >> "%LOGFILE%"
echo ============================== >> "%LOGFILE%"

robocopy "\\192.168.9.166\D\Data MMP\7. Store File" "C:\Users\Server\Documents\rijal\7. Store File" /E /XO /W:0 /R:0 /COPY:DAT /DCOPY:DAT /MT:16 /LOG+:"%LOGFILE%"

robocopy "\\192.168.9.166\D\Data MMP\9. Dyah File" "C:\Users\Server\Documents\rijal\9. Dyah File" /E /XO /W:0 /R:0 /COPY:DAT /DCOPY:DAT /MT:16 /LOG+:"%LOGFILE%"

echo Selesai pada %date% %time% >> "%LOGFILE%"</code></pre>

    <div class="ok">
      <strong>Perhatian</strong>: Format parsing <code>%date%</code> dan <code>%time%</code> bergantung regional Windows; sesuaikan jika perlu.
    </div>
  </section>

  <section>
    <h2>Menjadwalkan Backup dengan Task Scheduler</h2>
    <h3>Langkah singkat</h3>
    <ol>
      <li>Buka Task Scheduler: tekan <strong>Win + R</strong>, ketik <code>taskschd.msc</code>, tekan Enter.</li>
      <li>Pilih <strong>Create Basic Task</strong> atau <strong>Create Task</strong> untuk opsi lanjutan.</li>
      <li>Isi nama tugas misalnya <strong>Backup Robocopy</strong>.</li>
      <li>Pilih trigger <strong>Daily</strong> dan set waktu mulai <strong>14:00</strong> (jam 2 siang).</li>
      <li>Pilih action <strong>Start a program</strong> dan browse ke file <code>backup.bat</code>.</li>
      <li>Di tab <strong>General</strong>, centang <strong>Run whether user is logged on or not</strong> jika ingin berjalan tanpa sesi user.</li>
      <li>Centang <strong>Run with highest privileges</strong> jika script memerlukan hak Administrator.</li>
      <li>Simpan task dan masukkan kredensial akun jika diminta.</li>
    </ol>

    <h3>Pengaturan lanjutan yang disarankan</h3>
    <ul>
      <li>Gunakan akun service atau akun administrator yang memiliki akses ke UNC share.</li>
      <li>Jika UNC memerlukan kredensial, pastikan akun task memiliki akses atau buat mapped drive di skrip sebelum robocopy.</li>
      <li>Di tab <strong>Conditions</strong>, nonaktifkan opsi yang menghentikan task seperti <em>Start the task only if the computer is on AC power</em> jika tidak relevan.</li>
      <li>Di tab <strong>Settings</strong>, aktifkan <em>Allow task to be run on demand</em> dan atur restart jika gagal sesuai kebutuhan.</li>
    </ul>
  </section>

    <h2>Perbandingan Robocopy dan Rsync (Singkat)</h2>
    <table>
      <thead>
        <tr>
          <th>Aspek</th>
          <th>Robocopy</th>
          <th>Rsync</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Platform</td>
          <td>Windows native</td>
          <td>Linux/Unix native (bisa di Windows via WSL/Cygwin)</td>
        </tr>
        <tr>
          <td>Transfer remote</td>
          <td>Via SMB/UNC share</td>
          <td>Via SSH atau lokal</td>
        </tr>
        <tr>
          <td>Multi-threading</td>
          <td>Ya, /MT</td>
          <td>Tidak native</td>
        </tr>
        <tr>
          <td>Delta transfer</td>
          <td>Tidak (menyalin file penuh)</td>
          <td>Ya, bisa transfer hanya perubahan (efisien)</td>
        </tr>
        <tr>
          <td>Kontrol metadata</td>
          <td>Granular untuk NTFS (ACL, owner, audit)</td>
          <td>Fokus pada atribut file Unix</td>
        </tr>
      </tbody>
    </table>

    <p>Untuk sinkronisasi folder di jaringan Windows, Robocopy biasanya lebih praktis. Untuk transfer antar server via SSH atau transfer delta, rsync lebih unggul.</p>

  <section>
    <h2>Optimasi Performa Robocopy untuk Banyak Folder dan File</h2>

    <h3>Ringkasan singkat</h3>
    <p>Fokus optimasi: <strong>paralelisasi I/O</strong>, <strong>mengurangi overhead logging/console</strong>, dan <strong>menghindari opsi yang memperlambat</strong> (mis. /Z). Berikut tabel perbandingan opsi yang memengaruhi kecepatan.</p>

    <table>
      <thead>
        <tr>
          <th>Opsi</th>
          <th>Efek pada kecepatan</th>
          <th>Kapan dipakai</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>/MT[:n]</td>
          <td>Banyak mempercepat untuk banyak file kecil</td>
          <td>Gunakan untuk banyak file kecil; mulai /MT:16 lalu tuning</td>
        </tr>
        <tr>
          <td>/J</td>
          <td>Mempercepat untuk file besar (unbuffered I/O)</td>
          <td>File sangat besar (> GB); tidak kompatibel dengan /MT</td>
        </tr>
        <tr>
          <td>/IPG:n</td>
          <td>Memperlambat (menambah jeda)</td>
          <td>Gunakan hanya untuk mengurangi beban jaringan</td>
        </tr>
        <tr>
          <td>/Z, /ZB</td>
          <td>Lebih lambat (restartable mode)</td>
          <td>Gunakan jika jaringan tidak stabil dan perlu resume</td>
        </tr>
        <tr>
          <td>/NFL /NDL /NP /NJH /NJS</td>
          <td>Mempercepat dengan mengurangi output/log</td>
          <td>Selalu pakai untuk job besar agar overhead kecil</td>
        </tr>
        <tr>
          <td>/R: /W:</td>
          <td>Retry tinggi memperlambat jika banyak error</td>
          <td>Set /R:0 /W:0 untuk skip cepat pada file terkunci</td>
        </tr>
      </tbody>
    </table>

    <h3>Rekomendasi praktis</h3>
    <ol>
      <li>Gunakan <strong>/MT</strong> untuk banyak file kecil (mulai /MT:16 atau /MT:32).</li>
      <li>Gunakan <strong>/J</strong> untuk file sangat besar; jangan gabungkan dengan /MT.</li>
      <li>Kurangi output dengan <strong>/NFL /NDL /NP /NJH /NJS</strong>.</li>
      <li>Set retry rendah: <strong>/R:0 /W:0</strong> jika ingin proses cepat dan skip file terkunci.</li>
      <li>Hindari <strong>/Z</strong> kecuali perlu resume pada jaringan tidak stabil.</li>
      <li>Gunakan <strong>/LOG</strong> tapi ringkas; redirect ke file log daripada menampilkan daftar file di console.</li>
      <li>Uji konfigurasi pada subset data dan pantau CPU, disk queue, dan jaringan.</li>
    </ol>

    <h3>Contoh perintah dioptimalkan</h3>
    <p><strong>Banyak file kecil (paralel):</strong></p>
    <pre><code>robocopy "\\192.168.9.166\D\Data MMP\7. Store File" "C:\Users\Server\Documents\rijal\7. Store File" /E /XO /MT:16 /R:0 /W:0 /COPY:DAT /DCOPY:DAT /NFL /NDL /NP /NJH /NJS /LOG:C:\Logs\robocopy_store.log</code></pre>

    <p><strong>File sangat besar (unbuffered I/O):</strong></p>
    <pre><code>robocopy "\\192.168.9.166\D\Data MMP\LargeFiles" "D:\Backup\LargeFiles" /E /XO /J /R:1 /W:1 /COPY:DAT /DCOPY:DAT /NFL /NDL /NP /LOG:D:\Logs\robocopy_large.log</code></pre>

    <div class="warn">
      <strong>Catatan penting</strong>: Terlalu banyak thread (/MT terlalu tinggi) dapat menyebabkan I/O contention. Sesuaikan berdasarkan hasil uji dan pemantauan sumber daya.
    </div>
  </section>

  <section>
    <h2>Apakah 2 GB Termasuk File Besar untuk WLAN?</h2>
    <p>Secara praktis, <strong>2 GB tergolong file besar untuk transfer lewat WLAN</strong>, tetapi apakah terasa lama bergantung pada kecepatan Wi‑Fi, kondisi jaringan, dan perangkat keras.</p>

    <h3>Perkiraan waktu transfer (estimasi teoretis)</h3>
    <table>
      <thead>
        <tr><th>Throughput (Mbps)</th><th>Perkiraan waktu untuk 2 GB</th></tr>
      </thead>
      <tbody>
        <tr><td>20 Mbps</td><td>~13 menit</td></tr>
        <tr><td>50 Mbps</td><td>~5.3 menit</td></tr>
        <tr><td>100 Mbps</td><td>~2.7 menit</td></tr>
        <tr><td>300 Mbps</td><td>~55 detik</td></tr>
        <tr><td>600 Mbps</td><td>~28 detik</td></tr>
        <tr><td>1 Gbps</td><td>~16 detik</td></tr>
      </tbody>
    </table>

    <h3>Faktor yang memengaruhi kecepatan nyata</h3>
    <ul>
      <li>Standar Wi‑Fi (802.11n/ac/ax) dan kualitas sinyal.</li>
      <li>Interferensi, jarak, dan jumlah perangkat di jaringan.</li>
      <li>Overhead SMB/Windows dan enkripsi.</li>
      <li>Antivirus yang memindai file saat transfer.</li>
      <li>Kecepatan baca sumber dan tulis tujuan (HDD vs SSD).</li>
      <li>Pengaturan Robocopy (mis. /J untuk file besar, /MT untuk banyak file kecil).</li>
    </ul>

    <h3>Rekomendasi untuk transfer file besar lewat WLAN</h3>
    <ul>
      <li>Jika memungkinkan, gunakan kabel Ethernet untuk stabilitas dan kecepatan lebih baik.</li>
      <li>Untuk file besar, gunakan <code>/J</code> (unbuffered I/O): <code>robocopy ... /J ...</code>.</li>
      <li>Untuk jaringan tidak stabil, gunakan <code>/Z</code> atau <code>/ZB</code> agar bisa resume (tapi lebih lambat).</li>
      <li>Matikan atau kecualikan folder dari antivirus selama transfer untuk mengurangi overhead.</li>
      <li>Pertimbangkan pemindahan fisik (external HDD/SSD) jika jaringan terlalu lambat atau tidak andal.</li>
    </ul>
  </section>

  <section>
    <h2>Tambahan Praktis dan Troubleshooting</h2>
    <ul>
      <li>Jika UNC share memerlukan kredensial, jalankan task dengan akun yang memiliki akses atau buat skrip untuk memetakan drive sebelum robocopy.</li>
      <li>Gunakan <code>/TBD</code> jika share name belum tersedia dan Anda ingin menunggu definisi share.</li>
      <li>Untuk sinkronisasi penuh (termasuk menghapus file di destination yang tidak ada di source), gunakan <code>/MIR</code> dengan hati-hati (akan menghapus file di destination).</li>
      <li>Selalu uji skrip pada subset data sebelum menjalankan pada seluruh dataset produksi.</li>
    </ul>
